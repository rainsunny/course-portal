**课程时长**：第7天

## 开篇案例：同一个人，不同的领导方式

王经理是某科技公司的研发部经理，管理着一个12人的团队。团队里有不同背景、不同能力的员工，这让王经理常常感到困惑。

**小李**：新入职3个月的应届毕业生，计算机专业毕业，理论知识扎实，但实际项目经验不足。工作热情很高，什么都想尝试，但经常因为不知道如何下手而浪费时间。

> 王经理最初给小李完全的自主权，让他自己摸索。结果小李连续两周没有实质性产出，最后做出来的方案完全不符合要求。
> 
> 后来王经理改为给小李详细的指导，每天检查进度。结果小李感到压力很大，觉得自己不被信任，工作热情开始下降。

**老张**：入职8年的资深工程师，技术能力出众，是团队的技术骨干。但最近一年，老张的工作积极性明显下降，对新技术也不像以前那样热衷学习。

> 王经理尝试给老张更多的指导，详细说明每个任务的要求。老张的反应是："这些我都懂，不用你教。"
> 
> 王经理又尝试完全放手，让老张自己决定。结果老张觉得被忽视了，工作更加消极。

**思考**：如果你是王经理，你会如何分别领导这四位员工？

**案例启示**：
- 领导方式需要因人而异，这就是情境领导的核心思想
- 同一个团队中，不同成员处于不同的发展阶段
- 有效的领导者能够识别差异，灵活调整自己的领导风格

## 第一部分：领导与管理的区别

### 1.1 领导与管理的本质区别

**管理（Management）**：维持秩序，确保执行
**领导（Leadership）**：设定方向，激发变革

| 维度 | 管理 | 领导 |
|------|------|------|
| 焦点 | 维持秩序，确保执行 | 设定方向，激发变革 |
| 方式 | 控制和系统 | 愿景和感召 |
| 依赖 | 职位权力 | 个人影响力 |
| 时间视野 | 短期，注重当前 | 长期，注重未来 |
| 追求 | 效率、一致性 | 效果、变革 |

**案例对比**：

管理者说："我们要在本月底前完成这个项目。"
领导者说："我们要成为这个领域的最佳团队，这个项目是我们实现愿景的关键一步。"

### 1.2 领导与管理的平衡

**只有管理，没有领导**：
- 团队可能高效但缺乏活力
- 只知道完成命令，不知道为什么而战
- 创新能力弱，适应变化慢
- 员工主动性低，依赖指令

**只有领导，没有管理**：
- 团队可能有愿景但缺乏执行力
- 想法很多但落地很少
- 过程混乱，资源浪费
- 目标远大但难以实现

**最佳状态**：领导与管理相辅相成
- 用领导指明方向，激发热情
- 用管理确保执行，达成结果
- 在不同情境下灵活切换

## 第二部分：情境领导四步法

### 2.1 情境领导的核心思想

**核心观点**：没有一种领导方式是万能的，有效的领导者会根据不同的情况采取不同的领导风格。

### 2.2 四种领导风格

**S1 - 指令型（Directing）**
- **特点**：高指导行为，低支持行为
- **适用**：下属能力低、意愿低（R1）
- **做法**：领导者定义角色，告诉下属做什么、怎么做、何时做
- **典型语言**："按照这个步骤做"、"每天下班前向我汇报进展"

**S2 - 教练型（Coaching）**
- **特点**：高指导行为，高支持行为
- **适用**：下属有一定能力但意愿不高（R2）
- **做法**：领导者解释决策原因，鼓励双向沟通
- **典型语言**："我来解释一下为什么这样做"、"你觉得这个方案怎么样？"

**S3 - 支持型（Supporting）**
- **特点**：低指导行为，高支持行为
- **适用**：下属能力高但意愿低（R3）
- **做法**：领导者与下属共同决策，减少干预
- **典型语言**："你觉得应该怎么处理？"、"我能为你提供什么支持？"

**S4 - 授权型（Delegating）**
- **特点**：低指导行为，低支持行为
- **适用**：下属能力强、意愿高（R4）
- **做法**：领导者授权，让下属自主完成
- **典型语言**："这个项目就交给你了"、"有需要随时找我"

### 2.3 下属成熟度诊断矩阵

```
              │     能力低      │     能力高      
────────────┼────────────────┼────────────────
  意愿高    │   R2 教练型     │   R4 授权型     
            │   有热情但      │   能干且愿意    
            │   缺乏经验      │   充分授权      
────────────┼────────────────┼────────────────
  意愿低    │   R1 指令型     │   R3 支持型     
            │   新手或        │   能力够但      
            │   态度问题      │   缺乏动力      
```

### 2.4 情境领导实战案例

**案例背景**：陈经理是某互联网公司的产品部经理，团队中有四位不同特点的员工。

#### 案例1：小赵（R1 - 低能低愿）——使用S1指令型

**员工情况**：
- 新入职2个月，对产品业务不熟悉
- 之前在传统行业工作，缺乏互联网思维
- 对新工作环境不太适应，有些畏难情绪

**陈经理的做法**：
1. **明确任务**：详细说明每项任务的要求和步骤
2. **提供资源**：安排资深员工一对一辅导，提供产品文档模板
3. **密切跟进**：每天简短沟通，了解困难，及时纠正错误
4. **建立信心**：从小任务开始，逐步增加难度，及时肯定进步

**三个月后**：小赵已经能够独立完成基础工作，陈经理开始减少指导，转向教练型风格。

#### 案例2：小李（R2 - 低能高愿）——使用S2教练型

**员工情况**：
- 入职1年，工作热情高
- 学习能力强，但经验不足
- 经常主动承担超出能力范围的任务

**陈经理的做法**：
1. **双向沟通**："你为什么想接这个项目？"、"你觉得有哪些风险？"
2. **解释原因**："这个项目之所以重要，是因为..."
3. **引导思考**："如果遇到这个情况，你会怎么处理？"
4. **逐步授权**：开始时共同决策，逐步过渡到独立决策

**半年后**：小李的能力明显提升，陈经理开始使用支持型风格。

#### 案例3：老张（R3 - 高能低愿）——使用S3支持型

**员工情况**：
- 入职5年，技术能力强
- 近期工作积极性下降，经常抱怨
- 觉得工作没有挑战，发展受阻

**陈经理的做法**：
1. **倾听理解**："最近感觉你状态不太好，能跟我说说吗？"
2. **共同决策**："你觉得哪些工作更有意思？"
3. **提供支持**：为老张争取参加技术大会的机会，安排他负责创新项目
4. **情感连接**：定期一对一沟通，公开认可他的贡献

**两个月后**：老张恢复了工作热情，陈经理开始使用授权型风格。

#### 案例4：小王（R4 - 高能高愿）——使用S4授权型

**员工情况**：
- 入职3年，能力强，态度好
- 已经独立负责过多个项目
- 有明确的发展意愿，希望承担更多责任

**陈经理的做法**：
1. **充分授权**："这个新产品线就交给你负责了"
2. **资源支持**："需要什么资源随时告诉我"，帮助协调跨部门合作
3. **定期检查**：每月一次进展汇报，关注关键里程碑
4. **结果导向**：关注最终结果，而非过程细节

**结果**：小王带领团队完成了新产品线的开发和上线，获得了晋升机会。

### 2.5 情境领导的关键原则

#### 原则一：诊断先行
在选择领导风格之前，首先要诊断下属的成熟度：
- **能力评估**：专业技能、问题解决能力、工作经验
- **意愿评估**：工作热情、主动性、责任心

#### 原则二：灵活调整
下属的成熟度不是一成不变的，领导风格需要随之调整：
- 新任务可能需要降低风格（从S4到S2）
- 能力提升后需要提高风格（从S1到S4）
- 不同任务可能需要不同风格

#### 原则三：渐进发展
领导者的目标是帮助下属发展，使其能够接受更高程度的授权：
```
R1 ─────▶ R2 ─────▶ R3 ─────▶ R4
指令型    教练型    支持型    授权型
```

## 第三部分：变革型领导四维度

### 3.1 变革型领导的特征

变革型领导通过愿景、激励和赋能，超越追随者的期望。

**四个维度**：

| 维度 | 说明 | 管理者实践 |
|------|------|------------|
| **理想化影响力** | 成为追随者的榜样，有清晰的价值观和原则 | 以身作则，践行倡导的价值观 |
| **鼓舞性激励** | 有清晰的愿景，能够激发追随者的热情 | 描绘愿景，将日常工作与更大意义联系 |
| **智力激发** | 鼓励创新和挑战，容忍失败，鼓励学习 | 支持创新尝试，公开分享失败教训 |
| **个性化关怀** | 关注每个追随者的需求，辅导和支持成长 | 一对一沟通，制定个人发展计划 |

### 3.2 变革型领导 vs 交易型领导

| 维度 | 交易型领导 | 变革型领导 |
|------|------------|------------|
| 焦点 | 当前任务 | 未来愿景 |
| 方式 | 交换和奖惩 | 激励和赋能 |
| 员工 | 追随者 | 变革者 |
| 目标 | 完成任务 | 改变和成长 |

### 3.3 变革型领导实战案例

**案例背景**：张总是一家传统制造企业的CEO，面对数字化转型的压力，需要带领公司进行变革。

#### 一、理想化影响力——以身作则
张总没有只在会议上讲变革，而是：
- 自己率先学习数字化知识，每周安排固定时间研究新技术
- 公开承诺"不达变革目标，我第一个降薪"
- 在公司最困难的项目上亲力亲为，与员工一起加班
- 承认自己的不足："这方面你们比我懂，我要向你们学习"

**效果**：员工看到CEO真的在践行变革，而不是说说而已，开始相信变革是认真的。

#### 二、鼓舞性激励——描绘愿景
张总没有只强调危机和压力，而是：
- 在全员大会上描绘愿景："十年后，我们将成为智能制造的标杆企业"
- 讲述客户的成功故事，让员工看到变革的意义
- 设立阶段性目标："第一季度我们要上线数字化系统"
- 庆祝每一个小胜利：第一个数字化订单成功后，全公司庆祝

**效果**：员工从被动执行变为主动参与，开始为愿景而工作。

#### 三、智力激发——鼓励创新
张总没有只下达变革指令，而是：
- 设立"创新提案奖"，鼓励员工提出改进建议
- 组建跨部门创新小组，让年轻人主导
- 公开分享失败案例："这个方案失败了，但我们学到了..."
- 支持员工参加行业大会和培训

**效果**：公司涌现出大量创新提案，员工不再害怕提出想法。

#### 四、个性化关怀——关注个人
张总没有一刀切地推动变革，而是：
- 每月与核心员工一对一沟通，了解他们的想法和困难
- 为老员工提供转型培训，帮助他们适应新技术
- 为年轻员工提供晋升通道和项目负责机会
- 关注员工的工作压力，提供心理辅导资源

**效果**：不同背景的员工都感受到了被关注，变革阻力大大降低。

#### 变革成果
- 18个月后，公司成功上线数字化系统，效率提升30%
- 员工满意度从变革前的65%提升到82%
- 年轻员工流失率从25%下降到10%
- 创新提案从每年5个增加到每年50个

## 第四部分：情绪智力（EQ）四步法

### 4.1 情绪智力的四个维度

情绪智力是指识别、理解、管理自己和他人的情绪，并有效运用情绪信息来指导思维和行为的能力。

| 维度 | 定义 | 高表现 | 低表现 |
|------|------|--------|--------|
| **自我觉察** | 认识自己的情绪，了解情绪对行为的影响 | 准确识别情绪状态，明白情绪如何影响他人 | 不清楚为什么生气，被情绪控制后才意识到 |
| **自我管理** | 控制和调节自己的情绪，在压力下保持冷静 | 压力下保持冷静，能够抑制冲动反应 | 容易情绪爆发，压力下做出不理智决定 |
| **社会觉察** | 感知他人的情绪，理解他人的立场 | 准确解读他人情绪，理解他人立场 | 忽视他人情绪信号，难以理解他人观点 |
| **关系管理** | 有效沟通和表达，建立和维护关系 | 有效激励他人，善于处理冲突 | 难以建立深度关系，避免或激化冲突 |

### 4.2 情绪智力实战案例

**案例背景**：李经理是一家互联网公司的产品经理，负责一个10人的产品团队。最近，团队面临一个重要的项目上线，压力很大。

#### 情境一：面对自己的挫折

**事件**：项目上线前一周，测试发现一个严重bug，可能影响上线时间。李经理的第一反应是愤怒和焦虑。

**低情绪智力的反应**：
- 在团队面前大发雷霆："怎么到现在才发现这个问题！"
- 把责任推给测试人员
- 做出草率的决定："周末所有人加班"
- 情绪持续低落，影响后续判断

**高情绪智力的反应**：
1. **自我觉察**：意识到自己正在愤怒和焦虑
2. **自我管理**：深呼吸，暂停10分钟后再回应
3. **社会觉察**：考虑团队成员的感受——他们也压力很大
4. **关系管理**：召集团队，平静地分析问题

**李经理的话**：
> "我知道大家都压力很大。现在发现这个问题，确实让人沮丧。但我们先冷静下来，分析一下问题的严重性和解决方案。"

**结果**：团队齐心协力解决了问题，如期上线。团队没有因为压力而分崩离析，反而更加团结。

### 4.3 提升情绪智力的具体方法

#### 方法一：情绪日记（提升自我觉察）
```
时间：周一上午
情境：项目被质疑
情绪：愤怒
身体反应：心跳加速
行为反应：反驳对方
反思：是否反应过度？下次可以如何改进？
```

#### 方法二：暂停技巧STOP（提升自我管理）
- **S - Stop**：暂停，不要立即反应
- **T - Take a breath**：深呼吸
- **O - Observe**：观察自己的情绪和想法
- **P - Proceed**：选择适当的回应方式

#### 方法三：积极倾听（提升社会觉察）
- 全神贯注地听，不打断
- 注意对方的肢体语言和语调
- 用自己的话复述对方的观点
- 询问感受，而不仅仅是事实

#### 方法四：非暴力沟通（提升关系管理）
**观察** → **感受** → **需要** → **请求**

**低情商沟通**："你怎么总是迟到？太不负责任了！"

**非暴力沟通**：
> "我注意到这周你有三次迟到超过15分钟。（观察）
> 我有些担心，因为这会影响团队的工作安排。（感受）
> 我需要团队成员能够准时参与会议。（需要）
> 你能在以后准时参加会议吗？如果有什么困难，我们可以讨论。（请求）"

## 第五部分：教练式领导GROW模型

### 5.1 什么是教练式领导？

教练式领导是一种通过提问和倾听，帮助下属自我思考、发现问题、找到解决方案的领导方式。

**特点**：
- 不是直接给答案，而是引导思考
- 相信下属有解决问题的能力
- 关注长期发展和成长
- 建立平等、信任的关系

### 5.2 GROW模型四步法

```
G - Goal（目标）
│   你的目标是什么？
│   你想达成什么？
│   成功的标准是什么？
▼
R - Reality（现实）
│   目前的情况是什么？
│   已经尝试过什么？
│   障碍是什么？
▼
O - Options（选择）
│   你能做什么？
│   你有什么选择？
│   如果不考虑限制，你会怎么做？
▼
W - Way Forward（行动计划）
    你准备怎么做？
    什么时候开始？
    需要什么支持？
```

### 5.3 GROW模型实战对话

**背景**：王经理是某公司的产品部经理。他的下属小张最近负责一个重要项目，但进度滞后，团队士气低落。王经理决定用教练式对话帮助小张。

#### 第一阶段：Goal（目标）——明确目标
**王经理**："小张，今天我想和你聊聊这个项目。你希望今天讨论达成什么目标？"

**小张**："我希望能够找到解决项目滞后的办法，确保按时交付。"

**王经理**："好，按时交付是目标。那具体来说，成功完成这个项目意味着什么？"

**小张**："首先是按时上线，然后质量达标，客户满意。"

**王经理**："这些都很好。还有什么对你来说特别重要的吗？"

**小张**："其实...我还希望能够恢复团队的士气。最近大家压力很大，团队氛围不太好。"

**王经理**："所以你的目标是：按时交付高质量的产品，同时恢复团队士气。对吗？"

**小张**："是的，这正是我想达成的。"

#### 第二阶段：Reality（现实）——分析现状
**王经理**："现在让我们看看现状。目前项目的实际情况怎么样？"

**小张**："进度比计划落后两周，有三个核心功能还没完成。"

**王经理**："你已经尝试过什么方法来追赶进度？"

**小张**："我让大家加班，但效果不好。大家越加班，效率越低，抱怨也越多。"

**王经理**："你觉得导致项目滞后的主要原因是什么？"

**小张**："一开始需求变更太频繁，导致返工很多。后来我想要追赶进度，又安排了太多任务，团队消化不了。"

**王经理**："团队士气方面呢？你观察到了什么？"

**小张**："大家很少主动发言，会议上都是沉默。有人私下跟我说，感觉目标遥不可及，越来越没有信心。"

#### 第三阶段：Options（选择）——探索选择
**王经理**："现在我们来想想，要解决这些问题，有哪些可能的方案？"

**小张**："我觉得可以先砍掉一些非核心功能，集中精力把核心功能做好。"

**王经理**："好主意。还有呢？"

**小张**："也许可以申请延期一周？但这可能影响客户关系。"

**王经理**："这是个选项。还有其他可能吗？"

**小张**："我可以和团队坦诚沟通，让大家知道真实情况，一起想办法。"

**王经理**："这听起来也不错。现在让我们评估一下每个方案的利弊。"

**小张**：（思考后）
- 砍功能：优点是能够集中精力，但可能影响产品完整性
- 申请延期：可以保证质量，但有损信誉
- 团队沟通：能够恢复士气，但如果问题太大，沟通也解决不了

**王经理**："如果不考虑任何限制，你最想尝试哪个方案？"

**小张**："我想先试试砍功能和团队沟通。这样既能集中精力，又能恢复士气。"

#### 第四阶段：Way Forward（行动计划）——制定行动
**王经理**："好，我们来看看具体怎么做。你准备从哪里开始？"

**小张**："我打算明天上午召开一个团队会议，坦诚说明现状，和大家一起重新确定优先级。"

**王经理**："会议的目标是什么？"

**小张**："让团队了解真实情况，一起决定哪些功能可以推迟，哪些必须保留，然后重新制定计划。"

**王经理**："听起来很好。会议之后，你打算做什么？"

**小张**："我会和产品经理沟通功能删减的事情，争取他的支持。"

**王经理**："什么时候做？"

**小张**："会议当天下午。"

**王经理**："需要我提供什么支持吗？"

**小张**："如果产品经理不同意，可能需要您帮忙沟通。"

**王经理**："好的，我会支持你。现在，我们来总结一下行动计划："

| 时间 | 行动 | 预期结果 |
|------|------|----------|
| 明天上午 | 召开团队会议 | 达成共识，确定优先级 |
| 明天下午 | 与产品经理沟通 | 获得功能删减的支持 |
| 后天开始 | 按新计划执行 | 追赶进度 |
| 每周五 | 进度检查 | 确保按计划进行 |

**王经理**："你觉得这个计划可行吗？"

**小张**："可行。我觉得更有信心了。"

#### 对话效果
**一周后**：
- 团队会议效果很好，大家坦诚沟通后士气有所恢复
- 产品经理同意删减3个非核心功能
- 项目开始按新计划进行，进度逐渐恢复

**一个月后**：
- 项目如期上线，质量达标
- 团队士气恢复到7分
- 小张学会了用教练式方法管理团队

### 工具2：GROW教练对话模板

```
【Goal - 目标】
- 你想达成什么目标？
- 成功的标准是什么？
- 这个目标对你有什么意义？

【Reality - 现实】
- 目前的情况怎么样？
- 已经尝试过什么方法？
- 主要的障碍是什么？

【Options - 选择】
- 有哪些可能的解决方案？
- 每个方案的利弊是什么？
- 如果不考虑限制，你会怎么做？

【Way Forward - 行动】
- 你选择哪个方案？
- 具体什么时候开始？
- 需要什么支持？
- 如何衡量进展？
```

### 工具3：情绪智力STOP技巧

```
S - Stop（暂停）
   不要立即反应，给自己一点时间

T - Take a breath（深呼吸）
   深呼吸3次，让身体放松

O - Observe（观察）
   观察自己的情绪和想法
   问自己：我现在是什么情绪？
   这个情绪从哪里来？

P - Proceed（选择）
   选择适当的回应方式
   问自己：什么反应最有建设性？
```

## 常见误区警示

| 误区 | 正确认知 |
|------|----------|
| 一种风格走天下 | 有效的领导者会根据情境调整风格 |
| 领导力是天生的 | 领导力是可以通过学习和练习培养的 |
| 授权等于放任 | 授权需要设定检查点，提供必要支持 |
| 领导者不能示弱 | 适度的示弱和坦诚可以建立更深信任 |
| 情绪是软弱的表现 | 管理情绪是领导力的重要组成部分 |

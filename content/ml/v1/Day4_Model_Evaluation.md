# Day 4: 模型评估与统计推断

## 核心问题：如何知道模型真的"学会"了？

前三天我们学习了如何训练模型：定义损失函数、优化参数。但一个关键问题还没回答：**如何评估模型是否真的学到了规律，而不是死记硬背训练数据？**

这涉及机器学习的根本问题——**泛化能力**。今天我们深入探讨：

1. 训练/验证/测试的**统计意义**
2. 评估指标的**数学基础**
3. 如何判断模型表现的**统计显著性**

---

## 第一部分：数据分裂的统计意义

### 1.1 为什么不能只用训练数据评估？

**一个思想实验**：

假设老师给学生一份练习题（训练数据），学生反复练习直到全部做对。然后用**同样的题**考试，学生当然满分。但这能说明学生掌握知识了吗？

显然不能。只有用**新题**考试，才能检验学生是否真正理解。

**机器学习同样如此**：

- **训练数据**：练习题，用于学习
- **测试数据**：考试题，用于评估泛化能力

**核心问题**：模型在训练数据上的表现（训练误差）能代表在新数据上的表现（泛化误差）吗？

### 1.2 训练误差 vs 泛化误差

**定义**：

- **训练误差**：模型在训练数据上的误差
  $$\hat{R}_{train}(f) = \frac{1}{n_{train}}\sum_{i \in train} \mathcal{L}(y_i, f(x_i))$$

- **泛化误差（真实误差）**：模型在整个数据分布上的期望误差
  $$R(f) = \mathbb{E}_{(x,y) \sim P}[\mathcal{L}(y, f(x))]$$

**问题**：我们无法计算泛化误差（不知道真实分布），只能估计。

**估计方法**：使用独立于训练的数据（测试数据）计算测试误差：

$$\hat{R}_{test}(f) = \frac{1}{n_{test}}\sum_{i \in test} \mathcal{L}(y_i, f(x_i))$$

**关键假设**：测试数据与训练数据独立同分布（i.i.d.），测试误差是泛化误差的无偏估计。

### 1.3 数据分裂的数学基础

**为什么需要独立测试集？**

**定理（泛化误差界）**：以概率至少 $1-\delta$，对于所有函数 $f \in \mathcal{F}$：

$$R(f) \leq \hat{R}_{train}(f) + O\left(\sqrt{\frac{VC(\mathcal{F}) + \log(1/\delta)}{n}}\right)$$

其中 $VC(\mathcal{F})$ 是假设空间 $\mathcal{F}$ 的VC维度。

**含义**：
- 训练误差低估了真实误差
- 需要加上一个"复杂度惩罚项"
- 数据量越大，估计越准确

**问题**：如果用训练数据评估，我们只能得到训练误差，无法估计泛化能力。

### 1.4 三分数据集：训练、验证、测试

**三者的角色**：

| 数据集 | 用途 | 使用时机 | 模型是否"见过" |
|--------|------|----------|----------------|
| 训练集 | 学习参数 | 训练过程中 | 是 |
| 验证集 | 选择模型、调参 | 训练过程中 | 间接"见过" |
| 测试集 | 最终评估 | 模型选定后 | 否 |

**验证集的作用**：

在训练过程中，我们需要做出很多选择：
- 选择什么模型？（线性模型、决策树、神经网络？）
- 超参数如何设置？（正则化系数、学习率？）
- 何时停止训练？（早停？）

每次选择都基于验证集的表现。这意味着**验证集间接影响了模型选择**。

**测试集的独特地位**：

测试集应该只在**最终评估**时使用一次。它给出泛化能力的无偏估计。

**为什么测试集只能用一次？**

**信息泄露**：如果我们根据测试集表现调整模型，测试集就不再是独立数据。模型会"过拟合"到测试集。

### 1.5 数据分裂的比例

**常见比例**：

- 小数据集（<10000）：60% 训练 / 20% 验证 / 20% 测试
- 中等数据集（10000-1000000）：80% 训练 / 10% 验证 / 10% 测试
- 大数据集（>1000000）：98% 训练 / 1% 验证 / 1% 测试

**原则**：
- 训练集要足够大，保证学习效果
- 验证集要足够大，保证选择稳定
- 测试集要足够大，保证估计准确

**大数据时代的思考**：

当数据量很大时，1%的测试集可能已经足够（如100万样本的1%是1万样本）。此时更大的训练集更有价值。

### 1.6 交叉验证：充分利用数据

**问题**：数据量有限时，单次分裂浪费数据。

**K折交叉验证（K-Fold Cross-Validation）**：

1. 将数据分成K份
2. 每次用K-1份训练，1份验证
3. 重复K次，取平均

**数学表达**：

$$\hat{R}_{CV} = \frac{1}{K}\sum_{k=1}^{K} \hat{R}_k$$

其中 $\hat{R}_k$ 是第k折的验证误差。

**K的选择**：

- **K=5**：常用，计算量适中
- **K=10**：更精确，计算量更大
- **K=N（留一法）**：最精确，计算量最大

**交叉验证的偏差-方差权衡**：

| K值 | 偏差 | 方差 | 计算量 |
|-----|------|------|--------|
| 小（如2） | 高（训练数据少） | 低 | 小 |
| 大（如N） | 低（训练数据多） | 高 | 大 |

**为什么K大时方差高？**

每折的训练数据高度重叠，估计之间存在相关性，导致方差增加。

**什么时候用交叉验证？**

- 数据量有限
- 需要可靠的性能估计
- 模型选择或超参数调优

**注意**：交叉验证用于模型选择，仍需保留独立测试集用于最终评估。

## 第二部分：分类评估指标的数学基础

### 2.1 准确率的局限

**准确率（Accuracy）**：

$$\text{Accuracy} = \frac{\text{正确预测数}}{\text{总样本数}} = \frac{TP + TN}{TP + TN + FP + FN}$$

**问题**：准确率在**类别不平衡**时具有欺骗性。

**例子**：

疾病检测，1000人中有10人患病。如果模型预测所有人都健康：

- 准确率 = 990/1000 = 99%
- 但模型完全没有识别出患病者！

**结论**：准确率高不代表模型好。需要更细致的指标。

### 2.2 混淆矩阵：分类的完整图景

**定义**：

| | 预测正类 | 预测负类 |
|---|---------|---------|
| **实际正类** | TP (真正例) | FN (假负例) |
| **实际负类** | FP (假正例) | TN (真负例) |

**四个概念的本质**：

- **TP（True Positive）**：正确预测为正类的正类样本
- **FN（False Negative）**：错误预测为负类的正类样本（漏报）
- **FP（False Positive）**：错误预测为正类的负类样本（误报）
- **TN（True Negative）**：正确预测为负类的负类样本

**关键洞察**：

- FN 是"漏掉的正类"——本应识别，但没识别
- FP 是"误判的负类"——本不该识别，但错误识别

### 2.3 精确率与召回率

#### 精确率（Precision）

$$\text{Precision} = \frac{TP}{TP + FP} = \frac{\text{预测正确的正类}}{\text{预测为正类的总数}}$$

**含义**：预测为正类的样本中，有多少真的是正类？

**直觉**：模型说"是正类"时，有多少可信度？

**应用场景**：当假正类的代价很高时，关注精确率。

**例子**：垃圾邮件过滤
- FP：把重要邮件误判为垃圾邮件（代价高）
- 希望精确率高：被判定为垃圾邮件的，绝大部分真的是垃圾邮件

#### 召回率（Recall）

$$\text{Recall} = \frac{TP}{TP + FN} = \frac{\text{预测正确的正类}}{\text{实际正类的总数}}$$

**含义**：实际为正类的样本中，有多少被正确识别？

**直觉**：所有真正的正类，模型找到了多少？

**别名**：灵敏度（Sensitivity）、真正例率（TPR）

**应用场景**：当假负类的代价很高时，关注召回率。

**例子**：疾病诊断
- FN：漏诊患病者（代价高）
- 希望召回率高：患病的人几乎都被识别出来

### 2.4 精确率与召回率的权衡

**核心矛盾**：

- 提高精确率，通常降低召回率
- 提高召回率，通常降低精确率

**为什么？**

通过调整决策阈值来理解：

- **提高阈值**（更保守）：
  - 预测为正类的样本更少
  - 这些样本更可能是真正的正类 → 精确率提高
  - 但可能漏掉一些正类 → 召回率降低

- **降低阈值**（更激进）：
  - 预测为正类的样本更多
  - 覆盖更多真正的正类 → 召回率提高
  - 但混入更多负类 → 精确率降低

**图示**：

```
精确率 ↑
    |\
    | \      提高阈值
    |  \
    |   \
    |    *  (高精确率，低召回率)
    |     \
    |      \
    |       *  (平衡点)
    |        \
    |         *  (低精确率，高召回率)
    |          \
    +-----------→ 召回率
              降低阈值
```

### 2.5 F1分数：调和平均

**问题**：精确率和召回率都是重要指标，如何综合评估？

**算术平均**：

$$\frac{\text{Precision} + \text{Recall}}{2}$$

**问题**：如果一个指标很低，另一个很高，算术平均仍然可能很高。

**例子**：
- 精确率 = 1.0，召回率 = 0.01 → 算术平均 = 0.505
- 看起来还不错，但模型几乎没找到正类！

**调和平均（F1分数）**：

$$F_1 = \frac{2}{\frac{1}{\text{Precision}} + \frac{1}{\text{Recall}}} = \frac{2 \cdot \text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}$$

**调和平均的性质**：

- 对小值更敏感
- 只有当两个值都比较高时，F1才高
- 如果任何一个为0，F1为0

**例子**：
- 精确率 = 1.0，召回率 = 0.01 → F1 = 0.02
- 正确反映了模型性能很差

**为什么用调和平均？**

调和平均惩罚极端不平衡的情况，更符合直觉：**两个指标都要好**。

### 2.6 Fβ分数：加权调和平均

**更一般的定义**：

$$F_\beta = (1 + \beta^2) \cdot \frac{\text{Precision} \cdot \text{Recall}}{(\beta^2 \cdot \text{Precision}) + \text{Recall}}$$

**β的含义**：

- $\beta > 1$：更重视召回率
- $\beta = 1$：同等重要（标准F1）
- $\beta < 1$：更重视精确率

**常用选择**：

- **F1**：精确率和召回率同等重要
- **F2**：召回率更重要（如疾病筛查）
- **F0.5**：精确率更重要（如垃圾邮件过滤）

### 2.7 特异度与真正例率

#### 特异度（Specificity）

$$\text{Specificity} = \frac{TN}{TN + FP} = \frac{\text{预测正确的负类}}{\text{实际负类的总数}}$$

**含义**：实际为负类的样本中，有多少被正确识别？

**别名**：真负例率（TNR）

#### 真正例率（True Positive Rate, TPR）

$$\text{TPR} = \text{Recall} = \frac{TP}{TP + FN}$$

#### 假正例率（False Positive Rate, FPR）

$$\text{FPR} = \frac{FP}{FP + TN} = 1 - \text{Specificity}$$

**含义**：实际为负类的样本中，有多少被错误预测为正类？

**关系**：

$$\text{TPR} + \text{FNR} = 1$$
$$\text{TNR} + \text{FPR} = 1$$

## 第三部分：ROC曲线与AUC——全局视角

### 3.1 从阈值到曲线

**核心思想**：分类器输出概率，不同阈值对应不同的决策规则，产生不同的（FPR, TPR）点。

**阈值变化的影响**：

| 阈值 | 预测结果 | TPR | FPR |
|------|---------|-----|-----|
| 高（如0.9） | 很少预测正类 | 低 | 低 |
| 中（如0.5） | 平衡 | 中 | 中 |
| 低（如0.1） | 很多预测正类 | 高 | 高 |

**ROC曲线**（Receiver Operating Characteristic）：

横轴：FPR（假正例率）
纵轴：TPR（真正例率）

遍历所有可能的阈值，得到一条曲线。

### 3.2 ROC曲线的绘制

**算法**：

1. 按预测概率降序排列样本
2. 遍历每个概率值作为阈值
3. 计算该阈值下的（FPR, TPR）
4. 连接所有点

**代码示例**：

```python
import numpy as np
import matplotlib.pyplot as plt

def roc_curve(y_true, y_scores):
    """
    y_true: 真实标签 [0, 1, 1, 0, ...]
    y_scores: 预测概率 [0.1, 0.8, 0.6, 0.3, ...]
    """
    # 按分数降序排列
    thresholds = np.sort(y_scores)[::-1]
    thresholds = np.insert(thresholds, 0, np.inf)  # 添加无穷大阈值
    thresholds = np.append(thresholds, -np.inf)     # 添加负无穷阈值
    
    fprs, tprs = [], []
    
    for thresh in thresholds:
        y_pred = (y_scores >= thresh).astype(int)
        
        # 计算混淆矩阵
        TP = np.sum((y_pred == 1) & (y_true == 1))
        TN = np.sum((y_pred == 0) & (y_true == 0))
        FP = np.sum((y_pred == 1) & (y_true == 0))
        FN = np.sum((y_pred == 0) & (y_true == 1))
        
        # 计算TPR和FPR
        TPR = TP / (TP + FN) if (TP + FN) > 0 else 0
        FPR = FP / (FP + TN) if (FP + TN) > 0 else 0
        
        tprs.append(TPR)
        fprs.append(FPR)
    
    return np.array(fprs), np.array(tprs), thresholds
```

### 3.3 ROC曲线的解读

**完美分类器**：

- ROC曲线经过左上角（0, 1）
- FPR = 0, TPR = 1：完美识别所有正类，没有误报
- AUC = 1.0

**随机猜测**：

- ROC曲线是主对角线
- TPR = FPR：没有任何分辨能力
- AUC = 0.5

**实际分类器**：

- ROC曲线在对角线与左上角之间
- 越接近左上角，性能越好
- AUC ∈ (0.5, 1.0)

**ROC曲线下方的点代表什么？**

曲线下方（靠近右下角）的点意味着 FPR 较高时 TPR 较低，这是不好的性能。但ROC曲线本身是单调的，不会出现这种情况。

### 3.4 AUC的数学定义

**AUC**（Area Under ROC Curve）：ROC曲线下面积。

**定义1：积分形式**

$$\text{AUC} = \int_0^1 \text{TPR}(t) \, d(\text{FPR}(t))$$

**定义2：概率形式**（更直观）

$$\text{AUC} = P(\text{score}(\text{正样本}) > \text{score}(\text{负样本}))$$

**含义**：随机抽取一个正样本和一个负样本，正样本得分高于负样本的概率。

**证明等价性**：

考虑所有正负样本对。对于阈值 $t$：
- TPR(t) 是正样本得分 > t 的比例
- FPR(t) 是负样本得分 > t 的比例

ROC曲线下的面积等价于正样本得分高于负样本得分的概率。

**为什么AUC重要？**

1. **阈值无关**：评估分类器的整体能力，不受特定阈值影响
2. **类别不平衡鲁棒**：不受正负类比例影响
3. **排序质量度量**：衡量模型对样本排序的能力

### 3.5 AUC的计算

**梯形法则**：

$$\text{AUC} = \sum_{i=1}^{n-1} \frac{(\text{TPR}_i + \text{TPR}_{i+1})}{2} (\text{FPR}_{i+1} - \text{FPR}_i)$$

**概率形式计算**（更高效）：

```python
def auc_score(y_true, y_scores):
    """
    计算AUC：正样本得分高于负样本得分的概率
    """
    pos_scores = y_scores[y_true == 1]
    neg_scores = y_scores[y_true == 0]
    
    # 计算所有正负样本对的比较
    count = 0
    total = 0
    
    for pos in pos_scores:
        for neg in neg_scores:
            if pos > neg:
                count += 1
            elif pos == neg:
                count += 0.5  # 相等时贡献0.5
            total += 1
    
    return count / total
```

**优化版本**（利用排序）：

```python
def auc_score_fast(y_true, y_scores):
    """
    O(n log n) 复杂度的AUC计算
    """
    # 按分数排序
    sorted_indices = np.argsort(y_scores)[::-1]
    y_true_sorted = y_true[sorted_indices]
    
    # 计算正样本的排名和
    n_pos = np.sum(y_true)
    n_neg = len(y_true) - n_pos
    
    # 正样本的平均排名
    rank_sum = 0
    for i, label in enumerate(y_true_sorted):
        if label == 1:
            rank_sum += i + 1  # 排名从1开始
    
    # AUC公式
    auc = (rank_sum - n_pos * (n_pos + 1) / 2) / (n_pos * n_neg)
    return auc
```

### 3.6 ROC vs PR曲线

**PR曲线**（Precision-Recall Curve）：
- 横轴：召回率（Recall）
- 纵轴：精确率（Precision）

**什么时候用PR曲线？**

- **ROC**：类别平衡或关注整体性能
- **PR**：类别不平衡且关注正类性能

**原因**：

当负类远多于正类时：
- FPR = FP / (FP + TN)，TN 很大，FPR 变化很小
- ROC曲线可能过于乐观
- PR曲线更敏感于正类性能

**例子**：

1000个样本，10个正类，990个负类。

如果模型把所有样本都预测为负类：
- TPR = 0, FPR = 0 → ROC曲线上的 (0, 0) 点
- Precision = 0/0 未定义, Recall = 0 → PR曲线的起点

如果模型把100个样本预测为正类，其中8个是真的：
- TPR = 0.8, FPR = 92/990 ≈ 0.093 → ROC看起来还行
- Precision = 0.08, Recall = 0.8 → PR曲线显示问题严重

## 第四部分：统计显著性检验

### 4.1 为什么需要统计检验？

**问题**：模型A在测试集上的准确率是85%，模型B是84%。A真的比B好吗？

**可能的原因**：
1. A确实更好
2. 测试集的随机波动导致A碰巧得分更高

**统计检验**：判断差异是真实的还是随机的。

### 4.2 置信区间

**单个模型的置信区间**：

准确率的估计 $\hat{p} = \frac{X}{n}$，其中 $X$ 是正确预测数，$n$ 是测试样本数。

**中心极限定理**：当 $n$ 足够大时，$\hat{p}$ 近似服从正态分布：

$$\hat{p} \sim \mathcal{N}\left(p, \frac{p(1-p)}{n}\right)$$

**95%置信区间**：

$$\hat{p} \pm 1.96 \sqrt{\frac{\hat{p}(1-\hat{p})}{n}}$$

**例子**：

$n=1000$，$\hat{p} = 0.85$

$$CI = 0.85 \pm 1.96 \sqrt{\frac{0.85 \times 0.15}{1000}} = 0.85 \pm 0.022$$

即 $[0.828, 0.872]$。

**含义**：我们有95%的信心认为真实准确率在82.8%到87.2%之间。

### 4.3 两个模型的比较

**配对检验**（Paired Test）：

同一测试集上，两个模型的预测结果配对比较。

**定义**：
- $n_{00}$：两个模型都预测正确
- $n_{01}$：模型A正确，模型B错误
- $n_{10}$：模型A错误，模型B正确
- $n_{11}$：两个模型都预测错误

**McNemar检验**：

零假设 $H_0$：两个模型性能相同，即 $n_{01} = n_{10}$。

检验统计量（连续性校正）：

$$\chi^2 = \frac{(|n_{01} - n_{10}| - 1)^2}{n_{01} + n_{10}}$$

在 $H_0$ 下，$\chi^2$ 近似服从自由度为1的卡方分布。

**p值的解释**：
- p < 0.05：拒绝零假设，两个模型有显著差异
- p ≥ 0.05：不能拒绝零假设，差异可能由随机引起

### 4.4 交叉验证的统计检验

**问题**：交叉验证得到多个测试结果，如何比较两个模型？

**配对t检验**：

设模型A和B在K折交叉验证中的结果为 $\{a_1, ..., a_K\}$ 和 $\{b_1, ..., b_K\}$。

定义差异 $d_i = a_i - b_i$。

检验统计量：

$$t = \frac{\bar{d}}{s_d / \sqrt{K}}$$

其中 $\bar{d}$ 是差异均值，$s_d$ 是差异标准差。

在零假设下，$t$ 服从自由度为 $K-1$ 的t分布。

**注意事项**：

交叉验证的折之间存在相关性，违反了独立性假设。t检验可能过于乐观。

**5x2交叉验证检验**：

更稳健的方法，进行5次2折交叉验证，使用特殊的检验统计量。

## 第六部分：核心直觉总结

### 6.1 数据分裂的本质

**训练/验证/测试的分工**：

| 阶段 | 目的 | 信息流向 |
|------|------|---------|
| 训练 | 学习参数 | 数据 → 参数 |
| 验证 | 选择模型 | 数据 → 选择 |
| 测试 | 评估泛化 | 数据 → 性能 |

**关键原则**：测试数据必须在训练和选择过程中完全"隔离"，才能给出无偏估计。

### 6.2 评估指标的选择

| 场景 | 关注指标 | 原因 |
|------|---------|------|
| 类别平衡 | 准确率 | 简单直观 |
| 正类稀少 | PR曲线、F1 | 准确率误导 |
| 假阳性代价高 | 精确率 | 减少误报 |
| 假阴性代价高 | 召回率 | 减少漏报 |
| 阈值选择 | ROC曲线 | 全局视角 |

### 6.3 AUC的深刻含义

**AUC不是准确率，而是排序能力**：

$$\text{AUC} = P(\text{score}(\text{正样本}) > \text{score}(\text{负样本}))$$

**含义**：随机抽取一个正样本和一个负样本，模型能正确排序的概率。

**AUC的优势**：
- 与阈值无关
- 类别不平衡鲁棒
- 直接反映模型的"分辨能力"

### 6.4 统计检验的必要性

**两个模型准确率差1%，这差异真实吗？**

- 测试样本量100：可能只是随机波动
- 测试样本量10000：很可能是真实差异

**统计检验告诉我们**：观察到的差异有多大概率是真实的。

## 思考题

### 问题1：为什么测试集只能用一次？

<details>
<summary>点击查看提示</summary>

思考测试集的角色。如果根据测试集表现调整模型，测试集还是"独立"数据吗？
</details>

<details>
<summary>点击查看答案</summary>

**答案：多次使用测试集会导致信息泄露，测试误差不再是泛化误差的无偏估计。**

**问题的本质**：

测试集的作用是估计泛化误差——模型在从未见过的数据上的表现。关键假设是：**模型从未"见过"测试数据**。

**信息泄露的过程**：

1. 第一次使用测试集，得到误差 $E_1$
2. 发现误差较高，调整模型超参数
3. 第二次使用测试集，得到误差 $E_2 < E_1$
4. 继续调整...

**发生了什么？**

模型的超参数是根据测试集表现选择的。测试集间接"参与"了模型训练过程。模型开始"过拟合"到测试集。

**数学视角**：

设测试集大小为 $m$，我们尝试了 $k$ 种模型配置。

选择最优模型的测试误差：

$$\min_{i=1,...,k} E_i$$

这**低估**了真实泛化误差。因为我们选择了"运气最好"的配置。

**正确的做法**：

1. **验证集**：用于模型选择、超参数调优
2. **测试集**：只在最终评估时使用一次
3. 如果测试结果不理想，需要从头开始，收集新的测试数据

**实践中**：

- 学术论文：测试集只使用一次，报告这个结果
- 竞赛：测试集标签隐藏，最终排名基于测试集
- 工业界：保留一部分数据作为"最终测试"，永远不用于调参

**如果必须多次使用测试集？**

- 使用统计检验：确保改进不是随机波动
- 报告使用次数：让读者知道可能存在过拟合
- 使用交叉验证：每个fold的数据只作为测试集使用一次

**总结**：

测试集的"一次性"使用是保证泛化误差估计无偏的关键。多次使用意味着测试集不再是"未见数据"，评估结果失去意义。

</details>

### 问题2：准确率高但召回率低，在什么场景下是问题？

<details>
<summary>点击查看答案</summary>

**答案：当假负类（漏报）代价很高时，召回率低是严重问题，即使准确率很高。**

**场景分析**：

**场景1：疾病筛查（召回率至关重要）**

假设1000人中有10人患病：

- 模型预测所有人都健康
- 准确率 = 990/1000 = 99%
- 召回率 = 0/10 = 0%

**问题**：模型完全没识别出患病者！病人得不到治疗，后果严重。

**场景2：欺诈检测（召回率重要）**

- 假负类：漏掉欺诈交易，银行损失
- 假正类：正常交易被标记，用户不便
- 银行可能更愿意容忍一些不便，也不愿漏掉欺诈

**场景3：垃圾邮件过滤（精确率更重要）**

- 假正类：重要邮件被误判为垃圾邮件，错过重要信息
- 假负类：垃圾邮件进入收件箱，只是烦人
- 此时精确率更重要，召回率可以稍低

**量化分析**：

设：
- $C_{FN}$：漏报代价（假负类）
- $C_{FP}$：误报代价（假正类）

期望代价：
$$\text{Cost} = C_{FN} \cdot FN + C_{FP} \cdot FP$$

当 $C_{FN} \gg C_{FP}$（漏报代价远高于误报），应该优先提高召回率。

**决策阈值调整**：

- 降低决策阈值 → 召回率提高，精确率降低
- 提高决策阈值 → 精确率提高，召回率降低

**医疗诊断例子**：

设 $C_{FN} = 100$（漏诊代价），$C_{FP} = 10$（误诊代价，需进一步检查）。

最优阈值：
$$P(y=1|x) \geq \frac{C_{FP}}{C_{FP} + C_{FN}} = \frac{10}{10 + 100} \approx 0.09$$

远低于默认的0.5，意味着更激进地预测为阳性。

**总结**：

准确率是总体正确率，但不能反映特定错误类型的代价。当：
- 漏报代价高（疾病、欺诈）→ 关注召回率
- 误报代价高（垃圾邮件、警报）→ 关注精确率

</details>

### 问题3：为什么AUC对类别不平衡不敏感？

<details>
<summary>点击查看提示</summary>

思考AUC的定义：$P(\text{score}(\text{正样本}) > \text{score}(\text{负样本}))$。这个定义如何与类别比例相关？
</details>

<details>
<summary>点击查看答案</summary>

**答案：AUC衡量的是正负样本的排序能力，与类别比例无关。**

**AUC的概率定义**：

$$\text{AUC} = P(\text{score}(\text{正样本}) > \text{score}(\text{负样本}))$$

**关键洞察**：这个概率是在"一个正样本、一个负样本"的条件下计算的，不涉及类别比例。

**数学证明**：

设有 $n_+$ 个正样本，$n_-$ 个负样本。

所有正负样本对的数量：$n_+ \times n_-$

AUC可以计算为：

$$\text{AUC} = \frac{1}{n_+ \cdot n_-} \sum_{i=1}^{n_+} \sum_{j=1}^{n_-} \mathbf{1}[\text{score}(x_i^+) > \text{score}(x_j^-)]$$

**与类别比例的关系**：

分子是正负样本对的比较，分母是正负样本对的总数。类别比例 $n_+/n_-$ 不出现在公式中。

**对比准确率**：

$$\text{Accuracy} = \frac{TP + TN}{n_+ + n_-}$$

准确率涉及 $n_+$ 和 $n_-$，当 $n_- \gg n_+$ 时，准确率被负类主导。

**例子**：

1000个样本，10个正类，990个负类。

**模型1**：预测所有为负类
- 准确率 = 990/1000 = 99%
- AUC = 0.5（随机猜测水平，因为得分都相同）

**模型2**：完美分类
- 准确率 = 1000/1000 = 100%
- AUC = 1.0

**模型3**：识别出5个正类，但有5个假正类
- 准确率 = (5 + 985)/1000 = 99%
- 但仔细看：TP=5, FP=5, FN=5, TN=985
- 精确率 = 5/10 = 50%
- 召回率 = 5/10 = 50%
- AUC ≈ 0.5（如果排序混乱）到更高（取决于排序质量）

**AUC的计算过程**：

假设正样本得分：[0.9, 0.8, 0.7, ..., 0.1]（10个）
假设负样本得分：[0.09, 0.08, ..., 0.001]（990个）

AUC ≈ 1.0（几乎所有正样本得分都高于负样本）

即使负样本远多，AUC只关心"正样本是否比负样本得分高"。

**ROC曲线的构造**：

ROC曲线绘制时，FPR和TPR都是比率：
- TPR = TP / (TP + FN) = TP / n_+
- FPR = FP / (FP + TN) = FP / n_-

两者都归一化到[0,1]，不受类别比例影响。

**PR曲线不同**：

- 精确率 = TP / (TP + FP)
- 召回率 = TP / (TP + FN)

精确率的分母涉及FP，受负类数量影响。

**当负类很多时**：
- FP的基数很大，FP稍有变化，FP个数变化大
- 精确率敏感

**总结**：

AUC计算的是"正负样本对"的比较结果，与类别数量无关。这使得AUC在类别不平衡时仍能正确反映模型的分辨能力。

</details>

### 问题4：K折交叉验证中，K的选择如何影响偏差和方差？

<details>
<summary>点击查看答案</summary>

**答案：K小则偏差高、方差低；K大则偏差低、方差高。**

**偏差分析**：

**训练数据量**：

K折交叉验证中，每折用 (K-1)/K 的数据训练。

- K=2：每次用50%数据训练，训练数据少
- K=5：每次用80%数据训练
- K=10：每次用90%数据训练
- K=N（留一法）：每次用 (N-1)/N ≈ 100% 数据训练

**偏差来源**：

训练数据越少，模型学习越不充分，验证误差越高（高估真实误差）。

- K小：训练数据少 → 验证误差偏高 → 高偏差
- K大：训练数据多 → 验证误差接近真实 → 低偏差

**方差分析**：

**重叠度**：

K折之间训练数据高度重叠。

- K=2：两折的训练数据完全不重叠 → 估计独立 → 低方差
- K=N：每折只差一个样本 → 估计高度相关 → 高方差

**方差来源**：

估计之间的相关性越高，平均值的方差越大。

- K小：估计独立性强 → 低方差
- K大：估计相关性强 → 高方差

**权衡**：

| K | 偏差 | 方差 | 计算量 | 适用场景 |
|---|------|------|--------|---------|
| 2 | 高 | 低 | 低 | 数据量大 |
| 5 | 中 | 中 | 中 | 常用 |
| 10 | 低 | 中 | 高 | 数据量中等 |
| N | 最低 | 高 | 最高 | 数据量很小 |

**推荐**：

- **K=5或10**：经验上的最佳平衡
- **K=N**：数据量极小时（如<50样本）
- **K=2**：数据量很大或计算资源有限时

**实际考虑**：

1. **计算成本**：K越大，需要训练K次模型。如果模型训练慢，K不能太大。

2. **数据量**：
   - 数据量大：K=5足够，训练数据已接近全部
   - 数据量小：K=10或N，充分利用数据

3. **模型稳定性**：
   - 模型稳定（方差小）：K大一些
   - 模型不稳定（方差大）：K小一些

**分层K折（Stratified K-Fold）**：

对于类别不平衡问题，每折中正负类比例应与整体一致。使用分层K折交叉验证。

**总结**：

K折交叉验证的K选择是偏差-方差权衡的体现：
- K小：偏差高（训练数据少），方差低（估计独立）
- K大：偏差低（训练数据多），方差高（估计相关）
- 实践中K=5或10是经验最优值

</details>

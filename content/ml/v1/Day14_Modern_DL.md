## 一、预训练与微调范式

### 1.1 从零训练到预训练

**传统范式：从零训练**

```
流程：
├── 收集任务特定数据
├── 设计模型架构
├── 随机初始化参数
├── 训练直到收敛
└── 部署模型

问题：
├── 数据需求大
│   └── 需要大量标注数据
├── 训练成本高
│   └── 计算资源、时间
├── 泛化能力有限
│   └── 只学到任务特定知识
└── 不适合低资源场景
```

**现代范式：预训练+微调**

```
流程：
├── 预训练阶段
│   ├── 大规模无标注数据
│   ├── 自监督学习
│   └── 学习通用知识
├── 微调阶段
│   ├── 任务特定数据
│   ├── 有监督学习
│   └── 适配下游任务
└── 部署模型

优势：
├── 数据效率高
│   └── 少量标注数据即可微调
├── 训练成本低
│   └── 预训练模型已学到大量知识
├── 泛化能力强
│   └── 通用知识迁移到各任务
└── 低资源场景友好
```

### 1.2 预训练的本质

**为什么预训练有效？**

```
语言的本质：
├── 语法规则
├── 语义关系
├── 世界知识
└── 推理能力

预训练学习到：
├── 词法知识：词的语义、构词法

├── 语义知识：词义消歧、语义角色
├── 世界知识：常识、事实
└── 推理能力：逻辑、因果

这些知识是通用的，可以迁移到各种任务
```

**预训练任务的演化：**

```
早期：
├── Word2Vec：词向量

BERT时代：
├── 掩码语言模型（MLM）

GPT时代：
├── 因果语言模型（CLM）
├── 大规模数据
└── 学习生成能力

现代LLM：

```

### 1.3 微调策略

**全量微调：**

```python
# 加载预训练模型
model = AutoModelForSequenceClassification.from_pretrained('bert-base-uncased')

# 所有参数都参与训练
optimizer = AdamW(model.parameters(), lr=2e-5)

# 微调
for batch in train_dataloader:
    loss = model(**batch).loss
    loss.backward()
    optimizer.step()
```

```
特点：
├── 更新所有参数
├── 适应性强
├── 需要足够数据
└── 可能过拟合

适用场景：
├── 数据充足
├── 任务与预训练差异大
└── 追求最佳性能
```

**参数高效微调（PEFT）：**

```python
from peft import LoraConfig, get_peft_model

# LoRA配置
config = LoraConfig(
    r=8,  # 低秩矩阵的秩
    lora_alpha=32,
    target_modules=["q_proj", "v_proj"],
    lora_dropout=0.1
)

# 包装模型
model = get_peft_model(model, config)

# 只有少量参数需要训练
trainable_params = sum(p.numel() for p in model.parameters() if p.requires_grad)
total_params = sum(p.numel() for p in model.parameters())
print(f"可训练参数：{trainable_params} / {total_params} ({100*trainable_params/total_params:.2f}%)")
# 输出：可训练参数：0.1% - 1%
```

```
LoRA原理：
├── 冻结原始权重W
├── 添加低秩分解 W' = W + BA
│   └── B: d×r, A: r×d, r << d
├── 只训练A和B
└── 参数量大幅减少

优势：
├── 参数量减少99%+
├── 训练速度快
├── 内存占用低
├── 可以保存多个微调版本
└── 性能接近全量微调
```

**其他PEFT方法：**

```
1. Adapter
   ├── 在每层插入小型网络
   ├── 只训练Adapter参数
   └── 保持原始模型不变

2. Prefix Tuning
   ├── 添加可学习的前缀向量
   ├── 只训练前缀参数
   └── 输入层面的适配

3. Prompt Tuning
   ├── 添加可学习的提示向量
   ├── 只训练提示参数
   └── 更轻量的适配

4. BitFit
   ├── 只训练偏置参数
   ├── 参数量极少
   └── 简单有效
```

## 二、提示学习与少样本学习

### 2.1 提示学习的基本概念

**传统微调 vs 提示学习：**

```
传统微调：
预训练模型 → 任务特定头 → 监督训练 → 任务模型
├── 需要大量标注数据
├── 每个任务需要重新训练
└── 模型与任务紧密绑定

提示学习：
预训练模型 → 提示模板 → 直接预测
├── 无需或少量标注数据
├── 无需重新训练
└── 任务通过提示定义
```

**提示模板设计：**

```
情感分析任务：

输入：这部电影真好看！

传统方法：
├── 输入：这部电影真好看！
├── 模型：分类器
└── 输出：正面

提示方法：
├── 输入：这部电影真好看！这部电影是[MASK]的。
├── 模型：BERT预测[MASK]
└── 输出：[MASK]=好看/精彩/优秀 → 正面

模板设计：
├── 这部电影是[MASK]的。
├── 这部电影让我感到[MASK]。
├── 我对这部电影的评价是[MASK]。
└── 不同的模板效果可能不同
```

**零样本学习：**

```
无需标注数据的预测：

输入：这个产品质量很差。
提示：这个产品的评价是[MASK]的。
预测：差、糟糕、不好 → 负面

依赖预训练模型的知识：
├── 模型在预训练时见过大量文本
├── 学到了"质量差"→"负面"的关联
└── 通过提示激活这些知识
```

### 2.2 少样本学习

**Few-Shot Learning：**

```
定义：用极少样本（如1-5个）学习新任务

方法：
├── 上下文学习（In-Context Learning）
│   └── 在提示中提供示例
├── 元学习
│   └── 学习如何快速学习
└── 提示工程
    └── 设计有效的提示
```

**上下文学习示例：**

```
任务：情感分类

提示：
以下是电影评论的情感分类示例：

评论：这部电影太精彩了！
情感：正面

评论：浪费时间，不推荐。
情感：负面

评论：演员演技很好，剧情一般。
情感：正面

评论：这部电影真让我失望。
情感：[预测]

模型直接预测：负面
```

**为什么上下文学习有效？**

```
1. 模式识别
   ├── 模型识别输入-输出的模式
   ├── 根据模式推断新样本
   └── 类似人类的类比推理

2. 激活相关知识
   ├── 示例激活模型中的相关知识
   ├── 帮助模型"理解"任务
   └── 引导模型生成正确答案

3. 条件生成
   ├── 给定上下文，模型生成
   ├── 示例作为条件
   └── 影响生成的分布
```

### 2.3 提示工程

**提示设计原则：**

```
1. 清晰性
   ├── 明确任务目标
   ├── 避免歧义
   └── 提供必要的上下文

2. 结构化
   ├── 使用分隔符
   ├── 分步骤说明
   └── 清晰的输入输出格式

3. 示例驱动
   ├── 提供具体示例
   ├── 覆盖不同情况
   └── 展示期望的输出格式

4. 迭代优化
   ├── 测试不同提示
   ├── 分析失败案例
   └── 持续改进
```

**高级提示技术：**

```
1. 思维链（Chain of Thought）
   问题：小明有5个苹果，给了小红2个，又买了3个，现在有几个？
   
   普通提示：直接回答 → 6个
   
   CoT提示：让我们一步步思考：
   - 小明开始有5个苹果
   - 给了小红2个，剩下5-2=3个
   - 又买了3个，变成3+3=6个
   所以答案是6个。

2. 自我一致性（Self-Consistency）
   ├── 多次采样不同的推理路径
   ├── 投票选择最一致的答案
   └── 提高推理准确性

3. 思维树（Tree of Thoughts）
   ├── 探索多个推理分支
   ├── 评估每个分支
   └── 选择最优路径

4. ReAct（Reasoning + Acting）
   ├── 交替推理和行动
   ├── 可以调用外部工具
   └── 动态获取信息
```

## 三、指令微调与RLHF

### 3.1 指令微调

**为什么需要指令微调？**

```
预训练模型的问题：
├── 预训练目标：预测下一个词
├── 不理解指令
├── 无法按照用户意图回答
└── 可能生成不相关内容

例子：
用户：请解释量子力学
模型（未微调）：量子力学是一门学科...（续写）

用户：请解释量子力学
模型（指令微调）：量子力学是研究微观粒子运动规律的物理学分支...（回答）
```

**指令数据集构建：**

```
数据来源：
├── 人工标注
│   ├── 编写指令
│   ├── 编写回答
│   └── 质量控制
├── 现有NLP数据集改写
│   ├── 分类任务 → 指令形式
│   ├── 问答数据 → 指令形式
│   └── 翻译数据 → 指令形式
└── 模型自生成
    ├── 用强模型生成指令
    ├── 用强模型生成回答
    └── 筛选高质量样本

格式：
{
    "instruction": "请解释什么是机器学习",
    "input": "",
    "output": "机器学习是人工智能的一个分支..."
}
```

**指令微调过程：**

```python
# 指令微调数据
instruction_data = [
    {
        "instruction": "将以下句子翻译成英文",
        "input": "我爱学习",
        "output": "I love learning"
    },
    # 更多指令数据...
]

# 格式化提示
def format_instruction(item):
    if item["input"]:
        return f"指令：{item['instruction']}\n输入：{item['input']}\n输出："
    else:
        return f"指令：{item['instruction']}\n输出："

# 微调
for batch in dataloader:
    prompts = [format_instruction(item) for item in batch]
    targets = [item["output"] for item in batch]
    
    # 因果语言模型训练
    inputs = tokenizer(prompts, return_tensors="pt")
    labels = tokenizer(targets, return_tensors="pt")
    
    outputs = model(**inputs, labels=labels)
    loss = outputs.loss
    loss.backward()
    optimizer.step()
```

### 3.2 RLHF（人类反馈强化学习）

**RLHF的动机：**

```
指令微调的局限：
├── 回答质量难以评估
├── 缺乏细粒度反馈
├── 模型可能学会"说漂亮话"
└── 与人类真实偏好有差距

RLHF的想法：
├── 让人类对模型回答排序
├── 学习人类的偏好模型
├── 用偏好模型指导模型优化
└── 使模型输出更符合人类期望
```

**RLHF三步流程：**

```
第一步：有监督微调（SFT）
├── 用高质量指令数据微调
├── 获得基础的指令遵循能力
└── 模型可以按要求回答

第二步：训练奖励模型（RM）
├── 模型生成多个回答
├── 人类对回答排序
├── 训练奖励模型预测排序
└── 奖励模型学习人类偏好

第三步：强化学习优化（PPO）
├── 用奖励模型作为奖励函数
├── 用PPO算法优化语言模型
├── 平衡奖励和KL散度
└── 模型学会生成高质量回答
```

**奖励模型训练：**

```python
class RewardModel(nn.Module):
    def __init__(self, pretrained_model):
        super().__init__()
        self.model = pretrained_model
        self.value_head = nn.Linear(hidden_size, 1)
    
    def forward(self, input_ids):
        # 获取最后一层的隐藏状态
        outputs = self.model(input_ids)
        last_hidden = outputs.last_hidden_state[:, -1, :]
        
        # 预测奖励值
        reward = self.value_head(last_hidden)
        return reward

# 训练奖励模型
# 输入：同一提示的多个回答，人类排序
def train_reward_model(reward_model, ranked_responses):
    """
    ranked_responses: [response1, response2, ...]
    排序：response1 > response2 > ...（人类偏好）
    """
    rewards = [reward_model(r) for r in ranked_responses]
    
    # 对比损失：鼓励高质量回答得分高
    loss = 0
    for i in range(len(rewards) - 1):
        # response_i 应该比 response_{i+1} 得分高
        loss += -torch.log(torch.sigmoid(rewards[i] - rewards[i+1]))
    
    return loss
```

**PPO优化：**

```python
# 简化的PPO训练循环
for epoch in range(num_epochs):
    # 生成回答
    responses = model.generate(prompts)

    loss = -rewards.mean() + kl_penalty_weight * kl_penalty

```

### 3.3 DPO（直接偏好优化）

**DPO的动机：**

```
RLHF的问题：
├── 训练奖励模型需要大量数据
├── PPO训练不稳定
├── 超参数敏感
└── 实现复杂

DPO的想法：
├── 直接从偏好数据优化
├── 不需要训练奖励模型
├── 不需要PPO
└── 简化流程
```

**DPO损失函数：**

```
给定偏好对：(x, y_w, y_l)
├── x：提示
├── y_w：被偏好的回答
└── y_l：不被偏好的回答

DPO损失：
L = -E[log σ(β(log π_θ(y_w|x)/π_ref(y_w|x) - log π_θ(y_l|x)/π_ref(y_l|x)))]

其中：
├── π_θ：当前模型
├── π_ref：参考模型（初始模型）
├── β：温度参数
└── σ：sigmoid函数

直观理解：
├── 增加偏好回答的概率
├── 减少非偏好回答的概率
└── 保持参考模型的分布
```

## 四、深度学习最佳实践

### 4.1 模型训练技巧

**学习率调度的选择：**

```
1. Warmup + Cosine Decay
   ├── Transformer默认选择
   ├── Warmup：稳定初期
   ├── Cosine Decay：平滑衰减
   └── 适合大规模训练

2. One Cycle Policy
   ├── 快速上升到峰值
   ├── 然后缓慢下降
   └── 训练效率高

3. Constant + Decay
   ├── 前期保持恒定
   ├── 后期衰减
   └── 简单有效

选择建议：
├── 大模型：Warmup + Cosine
├── 中小模型：Constant + Decay 或 One Cycle
└── 微调：Constant 或 Linear Decay
```

**梯度累积：**

```python
# 显存不足时使用
accumulation_steps = 4

optimizer.zero_grad()
for i, batch in enumerate(dataloader):
    loss = model(**batch).loss
    loss = loss / accumulation_steps  # 归一化
    loss.backward()
    
    if (i + 1) % accumulation_steps == 0:
        optimizer.step()
        optimizer.zero_grad()
```

```
效果：
├── 等效于更大的batch size
├── 减少显存占用
└── 训练时间增加
```

**混合精度训练：**

```python
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()

for batch in dataloader:
    optimizer.zero_grad()
    
    with autocast():  # 自动混合精度
        loss = model(**batch).loss
    
    scaler.scale(loss).backward()  # 缩放梯度
    scaler.step(optimizer)
    scaler.update()
```

```
好处：
├── 减少显存占用约50%
├── 加速训练约2倍
└── 几乎不损失精度
```

### 4.2 调试和监控

**训练监控：**

```
必须监控：
├── Loss曲线
│   ├── 训练Loss：应该下降
│   ├── 验证Loss：应该下降
│   └── 差距：过大说明过拟合
├── 学习率
│   └── 确认调度正确
├── 梯度范数
│   ├── 过大：可能梯度爆炸
│   └── 过小：可能梯度消失
└── 准确率/指标
    └── 根据任务选择

可视化工具：
├── TensorBoard
├── Weights & Biases
└── MLflow
```

**常见问题排查：**

```
1. Loss不下降
   ├── 学习率过大/过小
   ├── 数据标签错误
   ├── 模型架构问题
   └── 梯度消失

2. Loss震荡
   ├── 学习率过大
   ├── batch size过小
   └── 数据增强过强

3. 过拟合
   ├── 数据量不足
   ├── 模型过大
   ├── 训练时间过长
   └── 解决：正则化、Dropout、早停

4. 训练慢
   ├── 数据加载瓶颈
   ├── 模型计算量大
   └── 优化：预取、混合精度、梯度累积
```

### 4.3 模型部署

**模型优化：**

```
1. 量化
   ├── FP32 → FP16/INT8
   ├── 模型大小减少50%-75%
   ├── 推理速度提升
   └── 精度轻微下降

2. 剪枝
   ├── 移除不重要的神经元
   ├── 减少模型大小
   └── 需要重新训练或微调

3. 蒸馏
   ├── 用大模型教小模型
   ├── 保留大部分性能
   └── 大幅减少参数量

4. ONNX导出
   ├── 统一的模型格式
   ├── 跨平台部署
   └── 优化推理速度
```

推理服务架构：

               ↓

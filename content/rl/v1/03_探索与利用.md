# Day 3：探索与利用的困境

> **核心问题**：在未知环境中，应该坚持已知的"好选择"，还是尝试未知的"可能性"？

## 引言：餐厅选择的两难

你来到一个新城市，准备吃晚餐。

**第一天**，你随机选了一家餐厅 A，味道不错，评分 8/10。

**第二天**，你面临选择：
- **利用（Exploit）**：再去餐厅 A，大概率还是 8/10
- **探索（Explore）**：尝试新餐厅 B，可能是 9/10，也可能是 3/10

这是一个经典的**探索-利用困境**：

- 只探索不利用：永远在尝试，从未真正享受
- 只利用不探索：错过更好的选择，陷入局部最优

**核心矛盾**：要获得最大长期收益，必须平衡"已知"与"未知"。

这个问题在强化学习中无处不在：选择哪个动作？尝试哪个策略？探索多远？

## 1. 多臂老虎机问题

### 1.1 问题定义

**多臂老虎机（Multi-Armed Bandit, MAB）**是探索-利用问题的最简化模型。

```
┌────────────────────────────────────────────┐
│                                            │
│    ┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐    │
│    │ 🎰  │  │ 🎰  │  │ 🎰  │  │ 🎰  │    │
│    │  1  │  │  2  │  │  3  │  │  K  │    │
│    └─────┘  └─────┘  └─────┘  └─────┘    │
│       ↑        ↑        ↑        ↑       │
│    μ₁=?    μ₂=?     μ₃=?     μₖ=?       │
│                                            │
│    每个臂有未知的中奖概率                    │
│    目标：最大化 T 次拉动的总收益             │
│                                            │
└────────────────────────────────────────────┘
```

**形式化定义**：

- K 个臂（动作），每个臂 i 有未知奖励分布，期望 μ_i
- 玩家依次选择臂，每次获得随机奖励 r_t ~ D_i
- 目标：T 次拉动后，最大化累积奖励

$$\text{目标} = \max \sum_{t=1}^{T} r_t$$

**关键约束**：不知道各臂的真实期望，必须通过尝试来学习。

### 1.2 为什么这是个难题？

**信息论视角**：

- 每次拉动提供一次"样本"，帮助估计该臂的期望
- 但每次拉动也是一次"机会成本"——本可以选择已知最好的臂

**统计学习视角**：

- 估计精度与样本量成正比：$\text{Var}(\hat{\mu}) \propto \frac{1}{n}$
- 但样本量 n 来自探索，探索本身有代价

**优化视角**：

- 全局最优需要知道所有臂的期望
- 但获取信息需要探索，探索可能选择次优臂

### 1.3 遗憾：衡量算法好坏的标准

直接比较累积奖励不方便（取决于具体实例），我们使用**遗憾（Regret）**：

$$\text{Regret}_T = T \cdot \mu^* - \sum_{t=1}^{T} r_t$$

其中 $\mu^* = \max_i \mu_i$ 是最优臂的期望。

**解释**：
- $T \cdot \mu^*$ 是"如果我们一直选最优臂"的理论最大收益
- 遗憾 = 理论最优 - 实际收益
- 好算法应该使遗憾随时间增长**尽可能慢**

**期望遗憾**：

$$\mathbb{E}[\text{Regret}_T] = T \cdot \mu^* - \mathbb{E}\left[\sum_{t=1}^{T} r_t\right] = \sum_{i=1}^{K} \Delta_i \cdot \mathbb{E}[n_i(T)]$$

其中：
- $\Delta_i = \mu^* - \mu_i$ 是臂 i 与最优臂的差距
- $n_i(T)$ 是 T 步内选择臂 i 的次数

**遗憾增长率**：
- 最坏情况：遗憾 O(T) —— 每步都选错
- 目标：遗憾 O(log T) 或更好

## 2. 经典探索策略

### 2.1 ε-贪心（ε-Greedy）

**核心思想**：以小概率探索，大概率利用。

**算法**：

```
以 ε 的概率：随机选择一个臂（探索）
以 1-ε 的概率：选择当前估计最好的臂（利用）
```

**实现**：

```python
import numpy as np

class EpsilonGreedy:
    def __init__(self, n_arms, epsilon=0.1):
        self.n_arms = n_arms
        self.epsilon = epsilon
        self.counts = np.zeros(n_arms)      # 每个臂的拉动次数
        self.values = np.zeros(n_arms)      # 每个臂的估计期望
    
    def select_arm(self):
        """选择臂"""
        if np.random.random() < self.epsilon:
            # 探索：随机选择
            return np.random.randint(self.n_arms)
        else:
            # 利用：选择估计值最大的臂
            return np.argmax(self.values)
    
    def update(self, arm, reward):
        """更新估计"""
        self.counts[arm] += 1
        n = self.counts[arm]
        # 增量更新均值
        self.values[arm] = self.values[arm] + (reward - self.values[arm]) / n
```

**遗憾分析**：

$$\mathbb{E}[\text{Regret}_T] \geq \frac{\epsilon T \Delta_{min}}{K}$$

问题：ε 固定时，遗憾随 T 线性增长！

**改进：衰减 ε**

$$\epsilon_t = \frac{c \cdot K}{t}$$

可以达到 O(log T) 遗憾。

### 2.2 UCB（Upper Confidence Bound）

**核心思想**：乐观估计不确定的选择。

**直觉**：
- 如果一个臂被拉动次数少，我们对它的估计不确定
- 不确定性高 = 可能被低估了
- 选择"上置信界"最高的臂

**UCB1 算法**：

$$a_t = \arg\max_i \left[ \hat{\mu}_i + c \sqrt{\frac{\ln t}{n_i}} \right]$$

其中：
- $\hat{\mu}_i$：臂 i 的经验均值
- $n_i$：臂 i 被拉动的次数
- $t$：总步数
- $c$：探索参数（通常 c=1）

**实现**：

```python
class UCB:
    def __init__(self, n_arms, c=1.0):
        self.n_arms = n_arms
        self.c = c
        self.counts = np.zeros(n_arms)
        self.values = np.zeros(n_arms)
        self.total_count = 0
    
    def select_arm(self):
        """选择臂"""
        # 确保每个臂至少被选一次
        for arm in range(self.n_arms):
            if self.counts[arm] == 0:
                return arm
        
        # UCB 公式
        ucb_values = self.values + self.c * np.sqrt(
            np.log(self.total_count) / self.counts
        )
        return np.argmax(ucb_values)
    
    def update(self, arm, reward):
        """更新估计"""
        self.counts[arm] += 1
        self.total_count += 1
        n = self.counts[arm]
        self.values[arm] = self.values[arm] + (reward - self.values[arm]) / n
```

**遗憾分析**：

$$\mathbb{E}[\text{Regret}_T] \leq \sum_{i: \mu_i < \mu^*} \left[ \frac{8 \ln T}{\Delta_i} \right] + \left(1 + \frac{\pi^2}{3}\right) \sum_{i=1}^{K} \Delta_i$$

**结论**：UCB 达到 O(K ln T) 遗憾，接近理论最优！

### 2.3 汤普森采样（Thompson Sampling）

**核心思想**：贝叶斯方法，从后验分布采样。

**贝叶斯框架**：
- 假设每个臂的奖励分布有一个参数 θ_i
- 维护 θ_i 的后验分布
- 每次选择时，从后验采样，选择采样值最大的臂

**伯努利老虎机（奖励 0/1）**：

假设奖励 ~ Bernoulli(θ_i)，使用 Beta 分布作为先验：

$$\text{Prior: } \theta_i \sim \text{Beta}(\alpha_i, \beta_i)$$

更新规则：
- 选择臂 i，获得奖励 r
- $\alpha_i \leftarrow \alpha_i + r$
- $\beta_i \leftarrow \beta_i + (1 - r)$

**实现**：

```python
class ThompsonSampling:
    def __init__(self, n_arms):
        self.n_arms = n_arms
        # Beta 分布参数，初始为 Beta(1, 1) = 均匀分布
        self.alpha = np.ones(n_arms)
        self.beta = np.ones(n_arms)
    
    def select_arm(self):
        """从后验分布采样，选择采样值最大的臂"""
        samples = np.random.beta(self.alpha, self.beta)
        return np.argmax(samples)
    
    def update(self, arm, reward):
        """更新后验分布"""
        if reward == 1:
            self.alpha[arm] += 1
        else:
            self.beta[arm] += 1
```

**遗憾分析**：

汤普森采样也能达到 O(K ln T) 遗憾，而且实践中往往表现优于 UCB！

**为什么汤普森采样有效？**

1. **自然探索**：不确定性高的臂，后验分布宽，采样波动大，有机会被选中
2. **自动调节**：随着数据增多，后验分布收窄，探索自然减少
3. **最优性**：在某些条件下被证明是最优的

## 3. 算法对比与实验

### 3.1 实验设置

```python
import matplotlib.pyplot as plt

class BanditEnvironment:
    """多臂老虎机环境"""
    
    def __init__(self, true_probs):
        """
        参数：
            true_probs: 每个臂的真实中奖概率
        """
        self.true_probs = np.array(true_probs)
        self.n_arms = len(true_probs)
        self.best_arm = np.argmax(true_probs)
        self.best_prob = np.max(true_probs)
    
    def pull(self, arm):
        """拉动臂，返回奖励（0 或 1）"""
        return np.random.random() < self.true_probs[arm]
    
    def get_regret(self, rewards):
        """计算遗憾"""
        optimal_reward = len(rewards) * self.best_prob
        return optimal_reward - sum(rewards)


def run_experiment(algorithm_class, env, n_rounds=1000, n_runs=100, **algo_kwargs):
    """运行实验"""
    all_regrets = []
    
    for run in range(n_runs):
        algo = algorithm_class(env.n_arms, **algo_kwargs)
        rewards = []
        
        for t in range(n_rounds):
            arm = algo.select_arm()
            reward = env.pull(arm)
            algo.update(arm, reward)
            rewards.append(reward)
        
        regret = env.get_regret(rewards)
        all_regrets.append(regret)
    
    return np.mean(all_regrets), np.std(all_regrets)


# 设置环境
true_probs = [0.3, 0.5, 0.7, 0.4, 0.6]  # 5 个臂，最优是第 3 个（0.7）
env = BanditEnvironment(true_probs)

# 运行实验
algorithms = [
    ('ε-Greedy (ε=0.1)', EpsilonGreedy, {'epsilon': 0.1}),
    ('ε-Greedy (ε=0.01)', EpsilonGreedy, {'epsilon': 0.01}),
    ('UCB', UCB, {'c': 1.0}),
    ('Thompson Sampling', ThompsonSampling, {}),
]

results = {}
for name, algo_class, kwargs in algorithms:
    mean_regret, std_regret = run_experiment(algo_class, env, n_rounds=1000, n_runs=100, **kwargs)
    results[name] = (mean_regret, std_regret)
    print(f"{name}: 平均遗憾 = {mean_regret:.2f} ± {std_regret:.2f}")
```

## 4. 遗憾理论：深入理解

### 4.1 遗憾的下界

**Lai-Robbins 下界（1985）**：

对于任何一致算法（consistent algorithm），遗憾满足：

$$\liminf_{T \to \infty} \frac{\mathbb{E}[\text{Regret}_T]}{\ln T} \geq \sum_{i: \mu_i < \mu^*} \frac{\Delta_i}{\text{KL}(\mu_i \| \mu^*)}$$

其中 $\text{KL}(p \| q)$ 是 KL 散度。

**意义**：
- 没有任何算法能比 O(ln T) 更好
- UCB 和汤普森采样达到了这个下界（忽略常数因子）
- 这是"最优"算法的数学定义

### 4.2 UCB 为什么有效？

**置信区间的视角**：

UCB 公式可以理解为：

$$\text{UCB}_i = \hat{\mu}_i + \underbrace{c \sqrt{\frac{\ln t}{n_i}}}_{\text{置信区间半径}}$$

**Hoeffding 不等式**：

$$P\left(|\hat{\mu}_i - \mu_i| > \epsilon\right) \leq 2e^{-2n_i \epsilon^2}$$

令 $\epsilon = \sqrt{\frac{\ln(2t^2)}{2n_i}}$，则：

$$P\left(|\hat{\mu}_i - \mu_i| > \sqrt{\frac{\ln(2t^2)}{2n_i}}\right) \leq \frac{1}{t^2}$$

**结论**：UCB 以高概率覆盖真实值，这解释了为什么它有效。

### 4.3 探索-利用的信息论视角

**信息价值（Value of Information）**：

探索的本质是获取信息。一条信息的价值 = 获得信息后收益的提升。

**熵减视角**：

- 探索减少对环境的不确定性（熵减）
- 利用使用已有信息获取收益
- 最优策略在"信息获取"和"收益获取"之间平衡

**形式化**：

设 $H(\theta)$ 为对环境参数 θ 的熵：
- 每次探索减少 $I(\theta; r_t | a_t)$ 的熵
- 但损失 $\mu^* - \mu_{a_t}$ 的即时收益
- 最优策略最大化长期信息价值

## 5. 从老虎机到强化学习

### 5.1 多臂老虎机 vs 强化学习

| 特性 | 多臂老虎机 | 强化学习 |
|------|------------|----------|
| 状态 | 无（或单状态） | 有，状态转移 |
| 动作 | 选择臂 | 选择动作 |
| 奖励 | 即时反馈 | 可能延迟 |
| 探索 | 随机选择臂 | 探索新状态/动作 |
| 利用 | 选择最优臂 | 遵循最优策略 |

**核心差异**：强化学习有**状态**，当前决策影响未来状态。

### 5.2 在强化学习中应用探索策略

**ε-贪心策略**：

$$\pi(a|s) = \begin{cases} 
1 - \epsilon + \frac{\epsilon}{|\mathcal{A}|} & \text{if } a = \arg\max_a Q(s,a) \\
\frac{\epsilon}{|\mathcal{A}|} & \text{otherwise}
\end{cases}$$

**UCB 在 RL 中**：

$$a_t = \arg\max_a \left[ Q(s_t, a) + c \sqrt{\frac{\ln N(s_t)}{N(s_t, a) + 1}} \right]$$

其中 $N(s)$ 是访问状态 s 的次数，$N(s, a)$ 是在状态 s 选择动作 a 的次数。

**汤普森采样在 RL 中**：

维护 Q 值的分布，每次从分布采样：
- 可用于基于模型的 RL
- 或深度 RL 中的参数探索

### 5.3 探索策略的代码示例

```python
class EpsilonGreedyAgent:
    """ε-贪心强化学习智能体"""
    
    def __init__(self, n_states, n_actions, epsilon=0.1, gamma=0.9, alpha=0.1):
        self.n_actions = n_actions
        self.epsilon = epsilon
        self.gamma = gamma
        self.alpha = alpha
        
        # Q 表
        self.Q = np.zeros((n_states, n_actions))
        # 访问计数（用于 UCB）
        self.N = np.zeros((n_states, n_actions))
    
    def select_action(self, state, method='epsilon_greedy'):
        """选择动作"""
        if method == 'epsilon_greedy':
            if np.random.random() < self.epsilon:
                return np.random.randint(self.n_actions)
            else:
                return np.argmax(self.Q[state])
        
        elif method == 'ucb':
            # 确保每个动作至少被选一次
            for a in range(self.n_actions):
                if self.N[state, a] == 0:
                    return a
            
            total = np.sum(self.N[state])
            ucb_values = self.Q[state] + np.sqrt(2 * np.log(total) / self.N[state])
            return np.argmax(ucb_values)
        
        elif method == 'thompson':
            # 简化版：假设 Q 值服从正态分布
            samples = np.random.normal(self.Q[state], 1.0 / (self.N[state] + 1))
            return np.argmax(samples)
    
    def update(self, state, action, reward, next_state, done):
        """更新 Q 值"""
        self.N[state, action] += 1
        
        if done:
            target = reward
        else:
            target = reward + self.gamma * np.max(self.Q[next_state])
        
        self.Q[state, action] += self.alpha * (target - self.Q[state, action])
```

## 6. 高级探索策略

### 6.1 置信区间上界变体

**UCB-V（考虑方差）**：

$$\text{UCB-V}_i = \hat{\mu}_i + \sqrt{\frac{2 V_i \ln t}{n_i}} + \frac{3 \ln t}{n_i}$$

其中 $V_i$ 是样本方差。

**KL-UCB**：

$$\text{KL-UCB}_i = \max\left\{ q : \text{KL}(\hat{\mu}_i \| q) \leq \frac{\ln t}{n_i} \right\}$$

更紧的置信区间，更好的实际表现。

### 6.2 信息论探索

**信息增益探索**：

选择能最大化信息增益的动作：

$$a_t = \arg\max_a \mathbb{E}_{r|a}[H(\theta) - H(\theta|r, a)]$$

**直觉**：选择最能减少不确定性的动作。

**实现复杂度高**，但理论性质好。

### 6.3 内在动机探索

**好奇心驱动**：

$$r_{\text{intrinsic}} = \| \phi(s_{t+1}) - \hat{\phi}(s_{t+1}) \|^2$$

其中 $\hat{\phi}$ 是预测下一状态的模型，$\phi$ 是状态表示。

**直觉**：选择预测误差大的状态，即"意外"的状态。

## 小结

**Day 3 核心要点**：

| 概念 | 定义 | 重要性 |
|------|------|--------|
| **探索-利用困境** | 平衡已知收益与未知可能 | ⭐⭐⭐⭐⭐ |
| **多臂老虎机** | 最简化的探索-利用模型 | ⭐⭐⭐⭐⭐ |
| **遗憾** | 衡量算法性能的标准 | ⭐⭐⭐⭐⭐ |
| **ε-贪心** | 简单有效的基线方法 | ⭐⭐⭐⭐ |
| **UCB** | 基于置信区间的乐观方法 | ⭐⭐⭐⭐⭐ |
| **汤普森采样** | 贝叶斯概率匹配 | ⭐⭐⭐⭐⭐ |
| **遗憾下界** | 理论最优性能极限 | ⭐⭐⭐⭐ |

**算法对比速查**：

| 算法 | 遗憾复杂度 | 需要调参 | 适用场景 |
|------|------------|----------|----------|
| ε-贪心 | O(T) 或 O(ln T)（衰减 ε） | 需要 | 简单场景 |
| UCB | O(K ln T) | 不需要 | 通用场景 |
| 汤普森采样 | O(K ln T) | 不需要 | 伯努利奖励 |

**核心洞察**：

> 探索是获取信息，利用是使用信息。最优策略在两者之间找到平衡点。UCB 和汤普森采样通过不同的数学框架实现了这种平衡——UCB 用置信区间，汤普森采样用后验分布。

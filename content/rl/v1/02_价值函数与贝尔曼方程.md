# Day 2：价值函数与贝尔曼方程

> **核心问题**：如何量化"一个状态有多好"？这个量化的数学结构是什么？

## 引言：导航中的智慧

假设你要从北京开车到上海，路线规划App给你三个选项：

```
路线 A：走高速，预计 5 小时
路线 B：走国道，预计 6 小时  
路线 C：混合路线，预计 5.5 小时
```

你会选哪条？如果只看时间，显然是 A。但如果：
- 路线 A 过路费 200 元
- 路线 B 免费
- 路线 C 过路费 50 元

你还会坚持选 A 吗？

这就是**价值评估**的核心问题：**如何综合多个因素，给出一个"综合评分"？**

在强化学习中，价值函数就是这样的评分系统——它告诉我们"从这个状态出发，按某个策略行动，最终能获得多少期望回报"。

而**贝尔曼方程**，则是这个评分系统背后的数学基石，被誉为强化学习的"牛顿定律"。

## 1. 价值函数的深入理解

### 1.1 为什么需要价值函数？

强化学习的目标是找到最优策略 π*。但什么叫"最优"？

**直观答案**：能获得最多奖励的策略。

**问题**：奖励是分散在各个时间步的，如何比较？

```
策略 A：[+1, +1, +1, +1, +1] → 总奖励 = 5
策略 B：[+10] → 总奖励 = 10
策略 C：[-5, -5, +20] → 总奖励 = 10
```

策略 B 和 C 总奖励相同，但体验完全不同。我们需要一个**统一的评估标准**。

**价值函数**就是答案：它把"未来的奖励流"压缩成"当前的一个数值"。

### 1.2 状态价值函数 V^π(s)

**定义**：从状态 s 出发，遵循策略 π 的期望累积回报。

$$V^\pi(s) = \mathbb{E}_\pi\left[\sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \bigg| S_t = s\right]$$

**逐层解析**：

1. **期望 E_π**：因为策略可能是随机的，环境转移也可能是随机的，所以要取期望
2. **累积回报 Σγ^k R_{t+k+1}**：未来所有奖励的加权和
3. **折扣因子 γ**：平衡即时与长远奖励

**物理意义**：
- V(s) 表示"站在状态 s，前景有多好"
- 高价值 = 好状态（接近目标、有利局面）
- 低价值 = 坏状态（危险、远离目标）

### 1.3 动作价值函数 Q^π(s, a)

**定义**：从状态 s 出发，执行动作 a，之后遵循策略 π 的期望累积回报。

$$Q^\pi(s, a) = \mathbb{E}_\pi\left[\sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \bigg| S_t = s, A_t = a\right]$$

**V 和 Q 的关系**：

$$V^\pi(s) = \sum_{a \in \mathcal{A}} \pi(a|s) Q^\pi(s, a)$$

**直觉**：
- V(s) = "这个位置有多好"（不管怎么走）
- Q(s, a) = "在这个位置走这一步有多好"

**哪个更有用？**
- V 简洁，但不直接告诉怎么做决策
- Q 更丰富，直接告诉每个动作的价值

### 1.4 最优价值函数

**最优状态价值函数**：

$$V^*(s) = \max_\pi V^\pi(s)$$

**最优动作价值函数**：

$$Q^*(s, a) = \max_\pi Q^\pi(s, a)$$

**为什么重要？**

如果我们知道了 Q*，最优策略就是：

$$\pi^*(s) = \arg\max_a Q^*(s, a)$$

**这就是强化学习的核心范式**：如果能计算出最优价值函数，最优策略唾手可得。

## 2. 贝尔曼方程：价值的递归结构

### 2.1 递归思想的威力

看一个简单的递归例子：**斐波那契数列**

$$F(n) = F(n-1) + F(n-2)$$

这个方程告诉我们：**当前值可以用之前的值表示**。一旦有了递归结构，就可以迭代计算。

贝尔曼方程的核心思想也是如此：**当前状态的价值 = 即时奖励 + 未来价值的折扣**。

### 2.2 贝尔曼期望方程

**对于 V^π**：

$$V^\pi(s) = \sum_a \pi(a|s) \sum_{s'} P(s'|s,a) \left[ R(s,a,s') + \gamma V^\pi(s') \right]$$

**解读**：

| 符号 | 含义 |
|------|------|
| $\sum_a \pi(a\|s)$ | 策略选择各动作的概率 |
| $\sum_{s'} P(s'\|s,a)$ | 环境转移到各状态的概率 |
| $R(s,a,s')$ | 即时奖励 |
| $\gamma V^\pi(s')$ | 折扣后的未来价值 |

**对于 Q^π**：

$$Q^\pi(s, a) = \sum_{s'} P(s'|s,a) \left[ R(s,a,s') + \gamma \sum_{a'} \pi(a'|s') Q^\pi(s', a') \right]$$

### 2.3 贝尔曼最优方程

**对于 V***：

$$V^*(s) = \max_a \sum_{s'} P(s'|s,a) \left[ R(s,a,s') + \gamma V^*(s') \right]$$

**对于 Q***：

$$Q^*(s, a) = \sum_{s'} P(s'|s,a) \left[ R(s,a,s') + \gamma \max_{a'} Q^*(s', a') \right]$$

**关键区别**：
- 贝尔曼期望方程：计算**给定策略**的价值
- 贝尔曼最优方程：计算**最优策略**的价值

### 2.4 数学推导：从定义到方程

让我们完整推导贝尔曼期望方程：

**Step 1**：展开回报的定义

$$V^\pi(s) = \mathbb{E}_\pi[G_t | S_t = s] = \mathbb{E}_\pi[R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots | S_t = s]$$

**Step 2**：分离即时奖励

$$= \mathbb{E}_\pi[R_{t+1} + \gamma (R_{t+2} + \gamma R_{t+3} + \cdots) | S_t = s]$$

$$= \mathbb{E}_\pi[R_{t+1} | S_t = s] + \gamma \mathbb{E}_\pi[G_{t+1} | S_t = s]$$

**Step 3**：利用全期望公式

$$\mathbb{E}_\pi[R_{t+1} | S_t = s] = \sum_a \pi(a|s) \sum_{s'} P(s'|s,a) R(s,a,s')$$

**Step 4**：利用马尔可夫性

$$\mathbb{E}_\pi[G_{t+1} | S_t = s] = \sum_a \pi(a|s) \sum_{s'} P(s'|s,a) \mathbb{E}_\pi[G_{t+1} | S_{t+1} = s']$$

$$= \sum_a \pi(a|s) \sum_{s'} P(s'|s,a) V^\pi(s')$$

**Step 5**：合并得到最终形式

$$V^\pi(s) = \sum_a \pi(a|s) \sum_{s'} P(s'|s,a) \left[ R(s,a,s') + \gamma V^\pi(s') \right]$$

### 2.5 图解贝尔曼方程

```
                    贝尔曼期望方程的"流向图"
                    
    ┌─────────────────────────────────────────────────────┐
    │                                                     │
    │   状态 s ──→ 策略 π ──→ 动作 a ──→ 环境 P ──→ 状态 s' │
    │     │         │         │         │         │      │
    │     │         │         │         │         │      │
    │     │         ↓         │         ↓         ↓      │
    │     │    π(a|s)    R(s,a,s')   Vπ(s')            │
    │     │         │         │         │              │
    │     │         └─────────┴─────────┘              │
    │     │                   │                        │
    │     ↓                   ↓                        │
    │   Vπ(s) = Σ π(a|s) Σ P(s'|s,a)[R + γVπ(s')]     │
    │                                                     │
    └─────────────────────────────────────────────────────┘
```

**贝尔曼最优方程的区别**：

```
                    贝尔曼最优方程的"流向图"
                    
    ┌─────────────────────────────────────────────────────┐
    │                                                     │
    │   状态 s ──→ 选择最优 ──→ 动作 a* ──→ 环境 P ──→ 状态 s' │
    │     │         max         │           │         │    │
    │     │                     │           │         │    │
    │     │                     │           ↓         ↓    │
    │     │                R(s,a*,s')    V*(s')          │
    │     │                     │           │            │
    │     │                     └───────────┘            │
    │     │                          │                  │
    │     ↓                          ↓                  │
    │   V*(s) = max_a Σ P(s'|s,a)[R + γV*(s')]         │
    │                                                     │
    └─────────────────────────────────────────────────────┘
```

## 3. 贝尔曼方程的应用

### 3.1 策略评估：计算 V^π

给定策略 π，如何计算 V^π？

**方法：迭代求解贝尔曼方程**

```python
def iterative_policy_evaluation(pi, P, R, gamma, theta=1e-6):
    """
    迭代策略评估
    
    参数：
        pi: 策略，pi[s][a] = 概率
        P: 转移概率，P[s][a][s'] = 概率
        R: 奖励函数，R[s][a][s'] = 奖励
        gamma: 折扣因子
        theta: 收敛阈值
    
    返回：
        V: 状态价值函数
    """
    V = {s: 0 for s in states}  # 初始化
    
    while True:
        delta = 0
        for s in states:
            v = V[s]
            # 贝尔曼期望方程
            V[s] = sum(
                pi[s][a] * sum(
                    P[s][a][s_prime] * (R[s][a][s_prime] + gamma * V[s_prime])
                    for s_prime in states
                )
                for a in actions
            )
            delta = max(delta, abs(v - V[s]))
        
        if delta < theta:
            break
    
    return V
```

### 3.2 策略改进：从 V^π 得到更好的策略

**策略改进定理**：如果对所有状态 s，新策略 π' 满足：

$$Q^\pi(s, \pi'(s)) \geq V^\pi(s)$$

则 π' 一定优于或等于 π，即 V^{π'}(s) ≥ V^π(s)。

**改进方法**：贪婪选择

$$\pi'(s) = \arg\max_a Q^\pi(s, a) = \arg\max_a \sum_{s'} P(s'|s,a) [R(s,a,s') + \gamma V^\pi(s')]$$

### 3.3 策略迭代：评估-改进循环

```
初始化任意策略 π₀

循环：
    1. 策略评估：计算 V^{π_k}
    2. 策略改进：π_{k+1} = greedy(V^{π_k})
    
直到 π_{k+1} = π_k（策略稳定）
```

### 3.4 价值迭代：直接求解最优价值

**核心洞察**：如果知道 V*，就不需要策略评估了！

**算法**：

```
初始化 V₀(s) = 0 对所有 s

循环：
    V_{k+1}(s) = max_a Σ P(s'|s,a)[R(s,a,s') + γV_k(s')]
    
直到 ||V_{k+1} - V_k|| < θ

输出策略：π*(s) = argmax_a Σ P(s'|s,a)[R(s,a,s') + γV*(s')]
```

## 4. 实践：完整的贝尔曼方程实现

### 4.1 定义一个标准 MDP 环境

```python
import numpy as np
from typing import Dict, List, Tuple

class MDP:
    """标准的马尔可夫决策过程环境"""
    
    def __init__(
        self, 
        states: List, 
        actions: List,
        transitions: Dict,  # P[s][a][s'] = probability
        rewards: Dict,      # R[s][a][s'] = reward
        gamma: float = 0.9
    ):
        self.states = states
        self.actions = actions
        self.P = transitions
        self.R = rewards
        self.gamma = gamma
        
    def get_transition_probs(self, s, a) -> List[Tuple]:
        """获取 (s, a) 的所有可能转移及概率"""
        return [(s_prime, self.P[s][a][s_prime], self.R[s][a][s_prime]) 
                for s_prime in self.states if self.P[s][a][s_prime] > 0]
    
    def is_terminal(self, s) -> bool:
        """判断是否终止状态"""
        return all(self.P[s][a][s] == 1.0 and self.R[s][a][s] == 0 
                   for a in self.actions)
```

### 4.2 策略评估的完整实现

```python
def policy_evaluation(mdp: MDP, policy: Dict, theta: float = 1e-6, max_iter: int = 1000):
    """
    策略评估：计算给定策略的价值函数
    
    参数：
        mdp: MDP 环境
        policy: 策略，policy[s][a] = 选择动作 a 的概率
        theta: 收敛阈值
        max_iter: 最大迭代次数
    
    返回：
        V: 状态价值函数
        iterations: 实际迭代次数
    """
    V = {s: 0.0 for s in mdp.states}
    
    for iteration in range(max_iter):
        delta = 0
        V_new = V.copy()
        
        for s in mdp.states:
            if mdp.is_terminal(s):
                continue
                
            v = 0
            for a in mdp.actions:
                action_prob = policy[s][a]
                for s_prime, trans_prob, reward in mdp.get_transition_probs(s, a):
                    v += action_prob * trans_prob * (reward + mdp.gamma * V[s_prime])
            
            delta = max(delta, abs(v - V[s]))
            V_new[s] = v
        
        V = V_new
        
        if delta < theta:
            print(f"策略评估收敛，迭代 {iteration + 1} 次")
            break
    
    return V, iteration + 1
```

### 4.3 策略改进

```python
def policy_improvement(mdp: MDP, V: Dict):
    """
    策略改进：根据价值函数改进策略
    
    参数：
        mdp: MDP 环境
        V: 状态价值函数
    
    返回：
        policy: 改进后的策略
        stable: 策略是否稳定（未改变）
    """
    policy = {s: {a: 0.0 for a in mdp.actions} for s in mdp.states}
    stable = True
    
    for s in mdp.states:
        if mdp.is_terminal(s):
            continue
        
        # 计算每个动作的价值
        action_values = {}
        for a in mdp.actions:
            q = 0
            for s_prime, trans_prob, reward in mdp.get_transition_probs(s, a):
                q += trans_prob * (reward + mdp.gamma * V[s_prime])
            action_values[a] = q
        
        # 找到最优动作
        max_q = max(action_values.values())
        best_actions = [a for a, q in action_values.items() if q == max_q]
        
        # 设置策略（等概率选择最优动作）
        for a in mdp.actions:
            new_prob = 1.0 / len(best_actions) if a in best_actions else 0.0
            if policy[s][a] != new_prob:
                stable = False
            policy[s][a] = new_prob
    
    return policy, stable
```

### 4.4 策略迭代

```python
def policy_iteration(mdp: MDP, max_iter: int = 100):
    """
    策略迭代：交替进行策略评估和策略改进
    
    参数：
        mdp: MDP 环境
        max_iter: 最大迭代次数
    
    返回：
        V: 最优价值函数
        policy: 最优策略
    """
    # 初始化随机策略
    policy = {s: {a: 1.0 / len(mdp.actions) for a in mdp.actions} for s in mdp.states}
    V = {s: 0.0 for s in mdp.states}
    
    for iteration in range(max_iter):
        # 策略评估
        V, _ = policy_evaluation(mdp, policy)
        
        # 策略改进
        policy, stable = policy_improvement(mdp, V)
        
        print(f"策略迭代第 {iteration + 1} 轮，策略稳定: {stable}")
        
        if stable:
            print(f"策略迭代收敛，共 {iteration + 1} 轮")
            break
    
    return V, policy
```

### 4.5 价值迭代

```python
def value_iteration(mdp: MDP, theta: float = 1e-6, max_iter: int = 1000):
    """
    价值迭代：直接计算最优价值函数
    
    参数：
        mdp: MDP 环境
        theta: 收敛阈值
        max_iter: 最大迭代次数
    
    返回：
        V: 最优价值函数
        policy: 最优策略
        iterations: 实际迭代次数
    """
    V = {s: 0.0 for s in mdp.states}
    
    for iteration in range(max_iter):
        delta = 0
        
        for s in mdp.states:
            if mdp.is_terminal(s):
                continue
            
            v = V[s]
            # 贝尔曼最优方程
            action_values = []
            for a in mdp.actions:
                q = 0
                for s_prime, trans_prob, reward in mdp.get_transition_probs(s, a):
                    q += trans_prob * (reward + mdp.gamma * V[s_prime])
                action_values.append(q)
            
            V[s] = max(action_values)
            delta = max(delta, abs(v - V[s]))
        
        if delta < theta:
            print(f"价值迭代收敛，迭代 {iteration + 1} 次")
            break
    
    # 提取最优策略
    policy = {s: {a: 0.0 for a in mdp.actions} for s in mdp.states}
    for s in mdp.states:
        if mdp.is_terminal(s):
            continue
        
        action_values = {}
        for a in mdp.actions:
            q = 0
            for s_prime, trans_prob, reward in mdp.get_transition_probs(s, a):
                q += trans_prob * (reward + mdp.gamma * V[s_prime])
            action_values[a] = q
        
        max_q = max(action_values.values())
        best_actions = [a for a, q in action_values.items() if q == max_q]
        
        for a in mdp.actions:
            policy[s][a] = 1.0 / len(best_actions) if a in best_actions else 0.0
    
    return V, policy, iteration + 1
```

```python
# 创建 MDP
mdp = MDP(states, actions, P, R, gamma=0.9)

# 运行价值迭代
print("=" * 50)
print("价值迭代")
print("=" * 50)
V_vi, policy_vi, iters_vi = value_iteration(mdp)

print("\n最优价值函数：")
for s in states:
    print(f"  V*({s}) = {V_vi[s]:.4f}")

print("\n最优策略：")
for s in states:
    if not mdp.is_terminal(s):
        best_actions = [a for a, p in policy_vi[s].items() if p > 0]
        print(f"  π*({s}) = {best_actions}")
```

## 5. 策略迭代 vs 价值迭代：深度比较

### 5.1 收敛性分析

| 算法 | 收敛保证 | 收敛条件 |
|------|----------|----------|
| 策略评估 | 收敛 | γ < 1 或有限 MDP |
| 策略迭代 | 收敛 | 有限 MDP，策略数有限 |
| 价值迭代 | 收敛 | γ < 1 或有限 MDP |

### 5.2 计算复杂度

对于 |S| 个状态，|A| 个动作的有限 MDP：

| 算法 | 每次迭代复杂度 | 迭代次数 |
|------|----------------|----------|
| 策略评估 | O(\|S\|²\|A\|) | 多次至收敛 |
| 策略迭代 | O(\|S\|²\|A\| × k) | k = 策略改进轮数 |
| 价值迭代 | O(\|S\|²\|A\|) | 多次至收敛 |

**经验法则**：
- 策略迭代通常迭代次数更少（策略改进快速收敛）
- 价值迭代每次迭代更快（不需要求解方程组）
- 对于小型 MDP，策略迭代往往更快

### 5.3 适用场景

```
策略迭代适合：
├── 状态空间较小
├── 转移矩阵稀疏
└── 需要精确策略

价值迭代适合：
├── 状态空间较大
├── 只需要价值函数
└── 后续需要结合函数逼近
```

### 5.4 对比实验

```python
def compare_algorithms(mdp: MDP):
    """比较策略迭代和价值迭代"""
    import time
    
    # 策略迭代
    start = time.time()
    V_pi, policy_pi = policy_iteration(mdp)
    time_pi = time.time() - start
    
    # 价值迭代
    start = time.time()
    V_vi, policy_vi, _ = value_iteration(mdp)
    time_vi = time.time() - start
    
    # 比较结果
    print("\n算法比较：")
    print(f"  策略迭代耗时: {time_pi:.4f}s")
    print(f"  价值迭代耗时: {time_vi:.4f}s")
    
    # 价值函数差异
    diff = max(abs(V_pi[s] - V_vi[s]) for s in mdp.states)
    print(f"  价值函数最大差异: {diff:.6f}")
    
    return {
        'policy_iteration': {'V': V_pi, 'policy': policy_pi, 'time': time_pi},
        'value_iteration': {'V': V_vi, 'policy': policy_vi, 'time': time_vi}
    }

# 运行比较
results = compare_algorithms(mdp)
```

## 6. 贝尔曼方程的理论保证

### 6.1 最优策略的存在性

**定理**：对于有限 MDP，至少存在一个最优策略 π*，使得对所有状态 s 和策略 π：

$$V^{\pi^*}(s) \geq V^\pi(s)$$

**证明思路**：
1. 状态价值函数空间是有限维的
2. 贝尔曼算子是压缩映射
3. 根据压缩映射定理，存在唯一不动点
4. 不动点对应最优价值函数

### 6.2 贝尔曼算子与压缩映射

定义**贝尔曼算子** T：

$$[TV](s) = \max_a \sum_{s'} P(s'|s,a) [R(s,a,s') + \gamma V(s')]$$

**性质**：T 是 γ-压缩映射

$$\|TV - TV'\|_\infty \leq \gamma \|V - V'\|_\infty$$

**意义**：
- 每次应用贝尔曼算子，价值函数之间的距离至少缩小 γ 倍
- 这保证了价值迭代的收敛性
- 收敛速度与 γ 相关：γ 越大，收敛越慢

### 6.3 收敛速度分析

**定理**：价值迭代第 k 步的误差

$$\|V_k - V^*\|_\infty \leq \frac{\gamma^k}{1-\gamma} \|V_1 - V_0\|_\infty$$

**实践意义**：
- 要达到误差 ε，需要迭代次数

$$k \geq \frac{\log(\epsilon(1-\gamma)/\|V_1 - V_0\|_\infty)}{\log\gamma}$$

- γ = 0.9 时，大约需要 100 次迭代达到 0.01 精度
- γ = 0.99 时，大约需要 1000 次迭代达到同样精度

| 概念 | 定义 | 重要性 |
|------|------|--------|

**一句话总结**：贝尔曼方程揭示了价值函数的递归结构，使我们可以通过迭代计算找到最优价值函数和最优策略。

**核心公式速查**：

| 方程 | 公式 |
|------|------|
| 贝尔曼期望 | $V^\pi(s) = \sum_a \pi(a\|s\|s) \sum_{s'} P(s'\|s,a)[R + \gamma V^\pi(s')]$ |
| 贝尔曼最优 | $V^*(s) = \max_a \sum_{s'} P(s'\|s,a)[R + \gamma V^*(s')]$ |
| 策略评估 | 迭代应用贝尔曼期望方程 |
| 策略改进 | $\pi'(s) = \arg\max_a Q^\pi(s, a)$ |
| 价值迭代 | 迭代应用贝尔曼最优方程 |
